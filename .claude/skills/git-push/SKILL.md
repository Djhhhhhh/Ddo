# GitPush 技能

自动完成 git 提交流程：暂存 → 生成提交信息 → 创建变更日志 → 提交并推送。

## 执行步骤

### 1. 检查 git 状态
使用 `git status` 检查当前仓库状态，确认是否有可提交的内容。

### 2. 暂存更改
如果有待提交的文件，执行 `git add -A` 暂存所有更改。

### 3. 获取变更信息
执行 `git diff --cached --stat` 获取暂存的变更统计，包括：
- 文件列表（新增、修改、删除）
- 代码行数变化（+/-）
- 用于生成提交信息和变更日志

### 4. 生成提交信息
根据变更内容生成简洁的提交信息，格式：`feat: 描述` / `fix: 描述` / `docs: 描述` / `chore: 描述`

### 5. 生成变更日志文件（暂不提交）
在 `docs/change-logs/` 目录下创建日志文件：
- 文件名格式：`YY-MM-DD-{简单描述}.md`
  * 简单描述取自 commit message 的关键词（前5个词或30字符）
  * 将特殊字符替换为连字符，保持文件名简洁
  * 示例：`25-04-07-feat-add-user-auth.md`
- 日志内容包含：
  * 提交信息、分支、日期时间
  * 变更文件列表（带状态：added/modified/deleted）
  * 变更统计（+/- 行数）

### 6. 暂存变更日志
执行 `git add docs/change-logs/` 将变更日志文件加入暂存区。

### 7. 执行提交并推送
使用生成的提交信息执行 `git commit -m "提交信息"`，然后立即执行 `git push` 一次性完成提交和推送。
同时捕获：日期时间、分支名、作者、文件列表。

## 输出格式

### 成功时

✓ 提交流程完成！
- 提交信息: xxx
- 变更文件: x 个新增, x 个修改, x 个删除
- 耗时: x 秒
- 远程推送: 已完成
- 变更日志: `docs/change-logs/YY-MM-DD-{描述}.md` ✅

### 失败时

✗ 提交失败
原因: [具体错误信息]

可能的原因：
- 无可提交的内容（工作区干净）
- 未连接到远程仓库
- 远程仓库拒绝推送（权限问题或冲突）
- 网络连接失败
- 变更日志创建失败（权限/路径问题）

### 无需提交时

✓ 工作区没有可提交的内容

## 注意事项

1. 始终使用 `git add -A` 暂存所有更改
2. 提交信息使用中文，简洁明了
3. 推送到当前分支的远程对应分支
4. 如果推送失败，尝试显示具体的 git 错误信息
5. 记录每个步骤的耗时，最后汇总报告

## 📝 变更日志格式

每个提交生成一个独立文件：
```
docs/change-logs/YY-MM-DD-{简单描述}.md
```

文件名生成规则：
- 格式：两位年份 + 两位月份 + 两位日期 + 短横线 + 描述
- 描述：从 commit message 提取前 5 个词或 30 字符，移除特殊符号
- 示例1：`25-04-07-feat-add-user-login.md`
- 示例2：`25-04-07-fix-reconnect-timeout.md`

文件内容模板：
```markdown
# 变更日志

**提交信息**: feat(auth): 用户登录功能
**分支**: main
**日期**: 2025-04-07
**作者**: Djhhh

## 变更文件
- auth.go (added)
- middleware.go (modified)
- config.yaml (modified)

## 统计
- 新增文件: 1
- 修改文件: 2
- 删除文件: 0
- 代码行数: +45 / -12

## 描述
<commit message 的简短说明或 AI 生成的摘要>
```
