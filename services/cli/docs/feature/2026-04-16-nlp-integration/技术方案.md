# CLI: NLP 集成 - 技术方案

## 背景与目标

根据 CLI MVP 需求文档，REPL 交互需要支持"自然语言模式"，将用户输入的文本通过 LLM 解析意图并执行对应操作。llm-py 服务已实现 `/api/nlp` 接口，提供意图识别和实体提取能力。本任务目标是在 CLI 侧实现与 llm-py NLP API 的集成，使 REPL 自然语言模式能够正确识别用户意图并路由到相应命令。

CLI MVP 中明确定义：
- 默认自然语言模式：直接输入任务描述，LLM 解析意图并执行
- 意图识别接口：POST `/api/v1/chat/nlp`

本方案将实现 CLI 侧的 NLP 集成层，完成从用户输入到意图解析的完整链路。

## 需求概述

CLI REPL 需要在自然语言模式下支持以下能力：

1. **意图识别**：将用户输入的自然语言发送到 llm-py `/api/nlp` 接口，获取 `intent`（如 `timer.create`, `kb.add`, `chat`）
2. **意图路由**：根据识别出的 intent，自动调用对应的 REPL 命令或系统命令
3. **参数提取**：从 NLP 响应中获取 `entities` 和 `parameters`，用于后续命令执行
4. **命令解析**：`/api/nlp/parse` 接口用于将自然语言命令解析为结构化参数

意图路由映射：
- `timer.create` -> 切换到 timer 模式，创建定时任务
- `timer.list` -> 切换到 timer 模式，列出定时任务
- `kb.add` -> 切换到 kb 模式，添加文档
- `kb.search` -> 切换到 kb 模式，搜索知识库
- `mcp.add` -> 切换到 mcp 模式，添加 MCP 配置
- `mcp.list` -> 切换到 mcp 模式，列出 MCP 配置
- `chat` -> 切换到 chat 模式，进行对话
- `status` -> 执行 status 命令，显示服务状态
- `help` -> 执行 help 命令，显示帮助

## 技术方案

### 方案选型

- **技术选型**：TypeScript + Fetch API（原生 HTTP 调用）
- **设计模式**：
  - **NLP Service 封装**：独立 `src/services/nlp.ts`，封装与 llm-py 的通信
  - **意图路由器**：独立的 `src/repl/intent-router.ts`，根据 intent 路由到对应处理函数
  - **响应式 REPL**：自然语言模式下，NLP 识别结果自动触发相应命令
- **依赖库**：无新增依赖，使用原生 `fetch` API

### 核心实现步骤

1. **创建 NLP Service**
   - 文件：`src/services/nlp.ts`
   - 功能：
     - `analyzeText(text: string, context?: object): Promise<NLPResponse>` - 调用 `/api/nlp`
     - `parseCommand(command: string, availableCommands: string[]): Promise<NLPParseResponse>` - 调用 `/api/nlp/parse`
   - 配置：读取 `config.yaml` 中的 `llm-py` 服务地址，默认 `http://127.0.0.1:8000`

2. **创建意图路由器**
   - 文件：`src/repl/intent-router.ts`
   - 功能：
     - `routeIntent(nlpResponse: NLPResponse): Promise<RouteAction>` - 根据 intent 返回路由动作
     - `createDefaultMode(modeManager: ModeManager, setMode: SetModeFunction)` - 创建默认模式的意图处理器
   - 路由表：维护 intent -> action 的映射

3. **修改默认模式 REPL 行为**
   - 文件：`src/repl/mode.ts` 或新建 `src/repl/modes/default-nlp.ts`
   - 功能：
     - 默认模式接收用户输入时，调用 NLP Service 进行分析
     - 根据分析结果调用意图路由器
     - 执行对应命令或切换模式

4. **添加配置读取**
   - 文件：`src/config/index.ts`（或扩展现有配置）
   - 功能：读取 llm-py 服务地址配置

5. **添加错误处理**
   - llm-py 服务不可用时，降级为直接转发到 chat 模式
   - 超时处理（默认 10 秒）
   - 网络错误友好提示

## 接口定义

### llm-py NLP 接口（已实现）

| 接口 | 方法 | 路径 | 请求参数 | 响应 |
|------|------|------|----------|------|
| 意图识别 | POST | `/api/nlp` | `{ text: string, context?: object, model?: string }` | `{ intent: string, confidence: float, entities: [], parameters: {}, reply: string }` |
| 命令解析 | POST | `/api/nlp/parse` | `{ command: string, available_commands: [], model?: string }` | `{ command: string, arguments: {}, is_ambiguous: bool, suggestions: [] }` |

### CLI 内部接口

| 接口 | 方法 | 路径 | 请求参数 | 响应 |
|------|------|------|----------|------|
| 意图分析 | 函数 | `nlpService.analyzeText()` | `text: string, context?: object` | `NLPResponse` |
| 命令解析 | 函数 | `nlpService.parseCommand()` | `command: string, availableCommands: string[]` | `NLPParseResponse` |
| 意图路由 | 函数 | `intentRouter.routeIntent()` | `NLPResponse` | `RouteAction` |
