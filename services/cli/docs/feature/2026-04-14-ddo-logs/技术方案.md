# CLI: ddo logs 实现 - 技术方案

## 背景与目标

根据 MVP 需求，`ddo logs` 命令需要提供查看各个服务日志的能力。目前已完成 `ddo init`、`ddo start`、`ddo stop`、`ddo status` 命令，logs 命令是 Phase 1 的最后一个任务。

需要实现的功能：
- 支持查看 cli、server-go、llm-py、web-ui 的日志
- 支持 `-f` 参数实时跟踪日志
- 支持 `-n` 参数指定显示行数
- 支持 `--since` 参数查看某时间之后的日志

## 需求概述

实现 `ddo logs [service]` 命令，功能包括：

1. **基础功能**：
   - 查看指定服务的日志文件内容
   - 无参数时默认显示 CLI 自身日志
   - 支持 `all` 参数同时显示所有服务日志（每个服务前加标识）

2. **过滤选项**：
   - `-n, --lines <number>`：显示最后 N 行（默认 100 行）
   - `-f, --follow`：实时跟踪日志（类似 `tail -f`）
   - `--since <time>`：显示指定时间之后的日志（如 "1h", "30m", "2024-01-01"）
   - `--level <level>`：按日志级别过滤（DEBUG, INFO, WARN, ERROR）

3. **服务列表**：
   - `cli`：CLI 自身日志
   - `server-go`：Go 后端服务日志
   - `llm-py`：Python LLM 服务日志
   - `web-ui`：Web 前端服务日志
   - `mysql`：MySQL 容器日志（通过 docker logs）
   - `all`：所有服务日志

4. **用户体验**：
   - 彩色输出区分不同日志级别
   - 自动高亮 ERROR 和 WARN 日志
   - 支持日志轮转后的文件检测

## 技术方案

### 方案选型

- **日志读取**：Node.js `fs` 模块 + `readline` 逐行读取
- **实时跟踪**：`fs.watchFile` 或 `fs.watch` 监听文件变化
- **时间解析**：自定义简单时间解析器（支持相对时间和 ISO 格式）
- **彩色输出**：使用 `chalk` 库（已存在依赖）
- **优雅退出**：Ctrl+C 信号处理，停止文件监听

### 核心实现步骤

1. **创建 logs.ts 命令文件**
   - 实现参数解析（service, lines, follow, since, level）
   - 验证服务名称有效性
   - 确定日志文件路径

2. **实现日志读取核心函数**
   - `readLogFile()`：读取文件最后 N 行
   - `readLogFileSince()`：读取指定时间之后的日志
   - `followLogFile()`：实时跟踪日志变化

3. **实现多服务日志显示**
   - `readAllLogs()`：聚合多个服务的日志
   - 按时间戳排序（如果日志包含时间戳）
   - 添加服务名前缀标识

4. **实现 MySQL 容器日志读取**
   - 调用 `docker logs` 命令
   - 支持相同的过滤参数

5. **添加彩色输出和高亮**
   - ERROR 红色高亮
   - WARN 黄色高亮
   - INFO 蓝色
   - DEBUG 灰色

6. **更新 index.ts 注册命令**
   - 添加完整的选项定义
   - 处理 Ctrl+C 信号

### 接口定义

| 选项 | 短选项 | 说明 | 示例 |
|------|--------|------|------|
| `--lines` | `-n` | 显示最后 N 行 | `-n 50` |
| `--follow` | `-f` | 实时跟踪 | `-f` |
| `--since` | | 显示某时间之后的日志 | `--since 1h` |
| `--level` | | 按级别过滤 | `--level ERROR` |

### 命令示例

```bash
# 查看 CLI 最后 100 行日志
ddo logs

# 查看 server-go 日志，实时跟踪
ddo logs server-go -f

# 查看最后 50 行 ERROR 级别日志
ddo logs llm-py -n 50 --level ERROR

# 查看过去 1 小时的日志
ddo logs --since 1h

# 查看所有服务日志（带服务前缀）
ddo logs all -n 20

# 查看 MySQL 容器日志
ddo logs mysql -f
```

## 核心数据结构

```typescript
// src/types/index.ts 新增
interface LogOptions {
  service?: string;           // 服务名称
  lines?: number;             // 显示行数（默认 100）
  follow?: boolean;           // 是否实时跟踪
  since?: string;             // 时间过滤（如 "1h", "30m"）
  level?: 'DEBUG' | 'INFO' | 'WARN' | 'ERROR';  // 日志级别过滤
}

interface LogEntry {
  timestamp?: Date;
  level?: string;
  service: string;
  message: string;
}
```
