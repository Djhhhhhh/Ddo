# CLI: ddo start 实现 - 技术方案

## 背景与目标

Ddo CLI 是项目的统一入口工具。`ddo init` 已完成基础环境初始化（目录结构、MySQL 容器）。接下来需要实现 `ddo start` 命令，用于：

- 启动所有后端服务（server-go、llm-py、web-ui）
- 实现进程管理（PID 文件记录）
- 确保服务按正确顺序启动
- 启动后进入 REPL 交互模式

## 需求概述

### 功能要求
1. **服务启动**：依次启动 server-go、llm-py、web-ui 三个后台服务
2. **进程管理**：为每个服务生成 PID 文件，记录在 `~/.ddo/services/` 目录
3. **启动顺序**：server-go 先启动（依赖 MySQL），然后是 llm-py，最后是 web-ui
4. **REPL 进入**：服务启动完成后自动进入 REPL 交互模式
5. **健康检查**：启动后验证各服务健康状态
6. **错误处理**：某个服务启动失败时给出清晰错误提示

### 验收标准
- 服务启动总时间 < 10 秒
- PID 文件正确记录进程 ID
- 支持优雅关闭（通过 PID 文件找到进程并终止）
- 跨平台兼容（Windows/macOS）

## 技术方案

### 方案选型

- **进程管理库**: Node.js 内置 `child_process` 模块，无需额外依赖
- **PID 文件格式**: 纯文本存储进程 ID，文件名为 `{service-name}.pid`
- **服务启动方式**: 子进程派生（`spawn`），保持后台运行
- **健康检查**: HTTP 请求轮询各服务的健康端点

### 核心实现步骤

1. **创建服务管理模块**
   - 实现 `src/services/manager.ts`：负责服务的启动、停止、状态查询
   - 实现 `src/services/pid-file.ts`：PID 文件的读写操作
   - 实现 `src/services/health-check.ts`：服务健康检查逻辑

2. **实现 ddo start 命令**
   - 读取配置获取各服务的启动命令和端口
   - 按顺序启动服务，每个服务启动后写入 PID 文件
   - 等待健康检查通过后再启动下一个服务
   - 全部启动成功后进入 REPL 模式

3. **实现进程管理功能**
   - 保存进程 ID 到 PID 文件
   - 提供根据 PID 文件查找和终止进程的方法
   - 支持优雅关闭信号（SIGTERM）

4. **添加错误处理和日志**
   - 服务启动失败时捕获错误并输出友好提示
   - 记录启动日志到 `~/.ddo/logs/cli.log`
   - 清理不完整的启动（如果某个服务失败，停止已启动的服务）

5. **编写测试用例**
   - 单元测试：PID 文件读写、健康检查逻辑
   - 集成测试：完整启动流程

## 接口定义

### 服务管理器接口

| 接口 | 方法 | 参数 | 返回 | 说明 |
|------|------|------|------|------|
| `startService` | async | `name: string, command: string[], options: SpawnOptions` | `Promise<{success: boolean, pid?: number, error?: string}>` | 启动单个服务 |
| `stopService` | async | `name: string` | `Promise<{success: boolean, error?: string}>` | 停止单个服务 |
| `getServiceStatus` | async | `name: string` | `Promise<{running: boolean, pid?: number}>` | 查询服务状态 |
| `startAll` | async | - | `Promise<{success: boolean, errors: string[]}>` | 启动所有服务 |
| `stopAll` | async | - | `Promise<{success: boolean, errors: string[]}>` | 停止所有服务 |

### PID 文件操作接口

| 接口 | 方法 | 参数 | 返回 | 说明 |
|------|------|------|------|------|
| `writePid` | sync | `serviceName: string, pid: number` | `void` | 写入 PID 文件 |
| `readPid` | sync | `serviceName: string` | `number \| null` | 读取 PID 文件 |
| `removePid` | sync | `serviceName: string` | `void` | 删除 PID 文件 |
| `isRunning` | async | `serviceName: string` | `Promise<boolean>` | 检查进程是否存活 |

### 健康检查接口

| 接口 | 方法 | 参数 | 返回 | 说明 |
|------|------|------|------|------|
| `waitForHealthy` | async | `url: string, timeout: number` | `Promise<boolean>` | 等待服务健康 |
| `checkHealth` | async | `url: string` | `Promise<boolean>` | 单次健康检查 |
