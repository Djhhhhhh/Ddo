# CLI: ddo init 实现 - 技术方案

## 背景与目标

Ddo CLI 需要一个初始化命令来一键准备工作环境。`ddo init` 是整个项目的入口点，负责创建必要的数据目录结构、生成配置文件、启动 MySQL Docker 容器，为后续所有功能奠定基础。

## 需求概述

实现 `ddo init` 命令，完成以下工作：
1. 创建数据目录结构 `~/.ddo/` 及各子目录
2. 生成 `docker-compose.yml` 用于 MySQL 容器
3. 生成 `config.yaml` 配置文件
4. 启动 MySQL Docker 容器
5. 验证数据持久化挂载正确

## 技术方案

### 方案选型

- 技术选型: Node.js + TypeScript，使用 `commander` 构建 CLI 框架
- 设计模式:
  - 命令模式：每个 CLI 命令独立成模块
  - 模板模式：配置文件使用模板生成
  - 策略模式：支持多种数据目录来源（环境变量、CLI 参数、默认值）
- 依赖库:
  - `commander` - 命令行参数解析
  - `chalk` - 终端彩色输出
  - `fs-extra` - 增强的文件系统操作
  - `yaml` - YAML 文件解析和生成
  - `docker-compose` - Docker Compose 执行（或调用 shell）

### 核心实现步骤

1. **目录结构初始化**
   - 解析数据目录路径（优先级：DDO_DATA_DIR > --data-dir > 默认 ~/.ddo）
   - 创建目录树：
     ```
     ~/.ddo/
     ├── config.yaml
     ├── docker/
     │   └── docker-compose.yml
     ├── services/       # PID 文件目录
     ├── data/mysql/     # MySQL 数据持久化
     ├── cache/          # 本地缓存
     ├── logs/           # 服务日志
     └── backup/         # 数据库备份
     ```

2. **配置文件生成**
   - 生成 `docker-compose.yml`：
     - MySQL 8.0 容器配置
     - 端口映射 3306:3306
     - Volume 挂载到本地 `data/mysql/`
     - 环境变量设置（root 密码、数据库名）
   - 生成 `config.yaml`：
     - 版本号
     - 数据目录路径
     - 服务端点配置（预留）
     - 日志级别

3. **MySQL 容器启动**
   - 检查 Docker Desktop 是否运行
   - 检查容器 `ddo-mysql` 是否已存在
   - 不存在则执行 `docker-compose up -d`
   - 等待 MySQL 健康检查通过
   - 输出成功/失败信息

4. **数据持久化验证**
   - 检查 Volume 挂载是否成功
   - 写入测试文件验证权限
   - 显示初始化完成摘要

## 接口定义

### CLI 命令格式

| 命令 | 参数 | 描述 |
|------|------|------|
| `ddo init` | `[--data-dir <path>]` | 初始化 Ddo 工作空间 |

### 配置接口

```typescript
interface DdoConfig {
  version: string;
  dataDir: string;
  database: {
    host: string;
    port: number;
    name: string;
    user: string;
  };
  logging: {
    level: 'debug' | 'info' | 'warn' | 'error';
    maxSize: string;
  };
}
```

### Docker Compose 配置

```yaml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    container_name: ddo-mysql
    ports:
      - "3306:3306"
    environment:
      MYSQL_ROOT_PASSWORD: ddo_root
      MYSQL_DATABASE: ddo
    volumes:
      - ~/.ddo/data/mysql:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping"]
      interval: 5s
      timeout: 3s
      retries: 5
```

### 手动验证步骤
1. 首次执行 `ddo init`，验证目录和文件生成
2. 检查 Docker 容器 `ddo-mysql` 运行状态
3. 验证 MySQL 数据写入：
   ```bash
   docker exec ddo-mysql mysql -uroot -pddo_root -e "CREATE TABLE test (id INT);"
   ls ~/.ddo/data/mysql/ddo/  # 验证表文件存在
   ```
4. 删除容器后重新 init，验证数据持久化
5. 使用 `--data-dir` 自定义路径验证

## 验收标准

- `ddo init` 执行成功率 > 95%
- MySQL 容器启动时间 < 30 秒
- 数据目录权限正确，无写入错误
- 重复执行幂等，不报错
