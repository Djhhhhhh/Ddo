# CLI: REPL 模式基础 - 技术方案

## 背景与目标

当前 CLI 已在 `src/repl/index.ts` 中实现了基础的 REPL 交互框架，但功能较为简单，仅支持基本的命令识别和提示信息。为了让 REPL 成为 CLI 的核心操作入口，需要进一步完善命令解析框架，包括：
- 支持命令参数解析（如 `/chat -m "hello"`）
- 支持子命令模式切换（如 `/kb`、`/timer`、`/mcp`）
- 实现统一的命令注册和分发机制
- 为后续 `/chat`、NLP 集成等功能提供坚实基础

## 需求概述

本任务要实现一个灵活、可扩展的 REPL 命令解析框架，具体包括：
1. 支持带参数的复杂命令解析
2. 实现子命令模式状态管理
3. 提供统一的命令注册机制
4. 优化用户交互体验（历史记录、Tab 补全提示）

## 技术方案

### 方案选型

| 方案 | 描述 | 选型说明 |
|------|------|----------|
| 自建解析器 | 使用正则或分词器手动解析 | 轻量、可控，适合 CLI 场景 |
| Commander.js | 现有的 Node CLI 库 | 太重，不适合 REPL 交互模式 |
| readline 原生 | Node.js readline 模块 | 基础支持，需配合自建解析 |

**最终选型**：自建轻量级命令解析器 + Node.js readline 模块
- 解析器使用简单的分词方式处理带引号的参数
- readline 提供历史记录和基础交互
- 架构清晰，易于扩展新命令

### 核心实现步骤

1. **创建命令解析器模块** (`src/repl/parser.ts`)
   - 实现分词函数，支持带引号字符串（如 `"hello world"`）
   - 解析命令名、选项（flags）和参数
   - 返回结构化的命令对象

2. **实现命令注册框架** (`src/repl/commands/index.ts`)
   - 定义 `Command` 接口，包含 name、description、handler 等属性
   - 创建命令注册表（Map 结构）
   - 实现 `registerCommand()` 和 `executeCommand()` 方法

3. **实现子命令模式管理** (`src/repl/mode.ts`)
   - 创建 `ReplMode` 枚举（Chat、Kb、Timer、Mcp）
   - 实现模式切换逻辑
   - 根据当前模式动态调整提示符（如 `ddo/kb>`）

4. **重构现有 REPL 入口** (`src/repl/index.ts`)
   - 集成新的解析器和命令框架
   - 将现有命令（/chat、/status、/exit 等）迁移到新的命令系统
   - 添加模式切换命令（/kb、/timer、/mcp）

5. **优化交互体验**
   - 添加命令自动补全提示（Tab 键）
   - 添加命令历史记录（上下箭头）
   - 优化错误提示和 help 命令输出

### 数据结构定义

```typescript
// 解析后的命令结构
interface ParsedCommand {
  name: string;           // 命令名（如 'chat'）
  args: string[];         // 位置参数
  flags: Record<string, string | boolean>; // 选项（如 -m, --model）
}

// 命令定义接口
interface ReplCommand {
  name: string;
  description: string;
  usage?: string;         // 使用示例
  aliases?: string[];     // 别名（如 'q' 对应 'exit'）
  handler: (ctx: CommandContext) => Promise<void>;
}

// 命令上下文
interface CommandContext {
  args: string[];
  flags: Record<string, string | boolean>;
  rl: readline.Interface;
  mode: ReplMode;
  setMode: (mode: ReplMode) => void;
}

// REPL 模式枚举
enum ReplMode {
  Default = 'default',
  Chat = 'chat',
  Kb = 'kb',
  Timer = 'timer',
  Mcp = 'mcp',
}
```

## 命令语法设计

```bash
# 基础命令
/chat hello world                      # 简单聊天
/chat -m "hello world"                 # 带标志参数
/chat --model gpt4 "hello"             # 长选项

# 子命令模式
/kb                                    # 进入知识库模式
> list                                 # ddo/kb> 提示符下直接输入子命令
> add <name> <path>
> search <query>

/timer                                 # 进入定时任务模式
> list
> add "0 9 * * *" "echo good morning"

/mcp                                   # 进入 MCP 管理模式
> list
> add <name> <url>

# 退出子命令模式
/exit                                  # 退出 REPL（任何模式下）
/back                                  # 返回默认模式（从子模式）
```
