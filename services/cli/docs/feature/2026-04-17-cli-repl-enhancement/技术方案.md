# CLI REPL 交互模式增强 - 技术方案

## 背景与目标

优化 CLI REPL 交互体验，使其更接近 Claude Code 的交互风格，同时增强知识库优先模式的切换机制。

### 需求概述

| 需求 | 描述 | 优先级 |
|------|------|--------|
| 1 | Shift+Tab 切换到知识库优先模式 | P0 |
| 2 | 取消重复显示用户输入 | P0 |
| 3 | AI 对话答复时 `ddo>` 换行，其他操作不换行 | P0 |

## 当前状态分析

### 键盘事件处理

当前实现 (`src/repl/index.ts`)：

```typescript
// Tab 键监听：切换知识库优先模式
process.stdin.on('keypress', (str: string, key: { name: string }) => {
  if (key.name === 'tab') {
    modeManager.toggleKbPriority();
    // ... 输出状态信息
  }
});
```

**问题**：
- 当前仅监听 `tab` 事件，无法区分 Shift+Tab 和 Tab
- 用户输入后会在下一行重复显示

### 输出模式

当前 `executeCommand` 统一在答复后换行：

```typescript
// 答复用户
console.log(chalk.blue('🤖 ') + response);
rl.prompt();  // 换行显示提示符
```

**问题**：
- 所有操作都换行，AI 对话和普通命令行为一致
- Claude Code 风格：AI 对话答复时换行，普通命令（如 /help）不换行

## 技术方案

### 需求 1: Shift+Tab 切换知识库优先模式

#### Step 1: 修改键盘事件监听逻辑

**目标文件**: `src/repl/index.ts`

```typescript
process.stdin.on('keypress', (str: string, key: { name: string; shift: boolean }) => {
  // Shift+Tab: 切换知识库优先模式
  if (key.name === 'tab' && key.shift) {
    modeManager.toggleKbPriority();
    rl.write('\r\n');
    const status = modeManager.kbPriorityMode ? '开启' : '关闭';
    console.log(chalk.magenta(`📚 知识库优先模式: ${status}`));
    rl.setPrompt(modeManager.getPrompt());
    rl.prompt();
    return;
  }

  // 普通 Tab: 保持原有的自动补全功能
  if (key.name === 'tab' && !key.shift) {
    // 现有的自动补全逻辑不变
    // ...
  }
});
```

**实现要点**：
- 通过 `key.shift` 属性判断是否按下了 Shift 键
- Shift+Tab 切换模式后直接返回，不触发自动补全
- 普通 Tab 保持原有行为

### 需求 2: 取消重复显示用户输入

#### Step 1: 移除输入回显

当前 `readline` 会自动回显用户输入。问题在于某些场景下输入被重复处理。

**可能原因分析**：
1. `rl.on('line')` 事件触发两次
2. 输入在某个地方被重复打印

**排查方向**：
- 检查 `completer` 回调是否有副作用
- 检查 `modeManager` 状态变化是否触发额外输出
- 检查是否存在调试日志

#### Step 2: 添加去重逻辑（保守方案）

**目标文件**: `src/repl/index.ts`

```typescript
let lastInput = '';
let lastInputTime = 0;
const INPUT_DEDUP_INTERVAL = 100; // ms

rl.on('line', async (input: string) => {
  const now = Date.now();

  // 去重：相同输入且间隔小于阈值
  if (input === lastInput && now - lastInputTime < INPUT_DEDUP_INTERVAL) {
    rl.prompt();
    return;
  }

  lastInput = input;
  lastInputTime = now;

  // ... 原有逻辑
});
```

### 需求 3: AI 对话答复换行，普通操作不换行

#### Step 1: 区分命令类型

**目标文件**: `src/repl/index.ts`

定义命令类型枚举：

```typescript
enum CommandType {
  Chat,      // AI 对话
  Command,   // 普通命令
}
```

修改 `executeCommand` 返回值结构：

```typescript
interface CommandResult {
  shouldContinue: boolean;
  outputType: CommandType;
  response?: string;  // 命令输出
}
```

#### Step 2: 根据输出类型决定是否换行

**目标文件**: `src/repl/index.ts`

```typescript
// 处理输入
rl.on('line', async (input: string) => {
  const trimmed = input.trim();

  if (trimmed === '') {
    rl.prompt();
    return;
  }

  // 解析命令
  const parsed = parseCommand(trimmed);

  // 如果有解析错误，显示警告
  if (parsed.errors.length > 0) {
    console.log(chalk.yellow('警告:'), parsed.errors.join(', '));
  }

  // 执行命令
  const result = await executeCommand(parsed, rl, modeManager);

  // 更新提示符（模式可能已改变）
  rl.setPrompt(modeManager.getPrompt());

  if (result.shouldContinue) {
    // 根据输出类型决定是否换行
    if (result.outputType === CommandType.Chat && result.response) {
      // AI 对话答复：先换行再显示提示符
      rl.write('\r\n');
    }
    // 普通命令：直接显示提示符，不额外换行
    rl.prompt();
  }
});
```

#### Step 3: 修改各命令返回正确的输出类型

**目标文件**: `src/repl/commands/chat.ts`

```typescript
export const chatCommand: ReplCommand = {

  description: '与 AI 对话',
  aliases: [],

```

**目标文件**: `src/repl/commands/help.ts` 等普通命令

```typescript
export const helpCommand: ReplCommand = {
```

## 影响范围

| 文件 | 修改内容 |
|------|----------|
| `src/repl/index.ts` | 键盘事件监听、输出控制、去重逻辑 |
| `src/repl/commands/index.ts` | 新增 `CommandResult` 类型和 `CommandType` 枚举 |

| `src/repl/commands/help.ts` | 返回 `CommandType.Command` |
| `src/repl/commands/*.ts` | 所有命令文件统一返回正确的 `outputType` |

## 验收标准

1. **Shift+Tab 测试**：
   - 在任意模式下按 Shift+Tab，切换知识库优先模式状态
   - 普通 Tab 仍触发自动补全，不切换模式

2. **输入去重测试**：
   - 连续快速输入相同内容，不重复处理
   - 正常输入间隔不受影响

3. **换行行为测试**：
   - `/chat 你好` → AI 答复后 `ddo>` 换行显示
   - `/help` → 帮助信息后 `ddo>` 不换行显示在同一行末尾
   - `/kb list` → 列表后 `ddo>` 不换行显示

## 风险与回滚

- **风险**：键盘事件捕获可能受终端类型影响，需在 Windows/macOS/Linux 多端测试
- **回滚**：修改仅限于 CLI 层，不影响 server-go 和 llm-py 服务
