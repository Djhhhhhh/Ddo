# server-go: MCP 管理 API - 技术方案

## 背景与目标

根据 MVP 文档，server-go 需要提供 MCP（Model Context Protocol）管理能力，支撑 CLI 和 Web UI 对 MCP Server 的配置、连接测试和工具调用需求。

当前已完成：
- MCPConfig 模型定义（`internal/db/models/mcp.go`）
- GORM MySQL 连接（p2-12）

本任务实现：
- MCP 配置 CRUD API
- MCP 连接池管理
- MCP 连接测试接口

## 需求概述

### 功能范围（P0）

1. **MCP 配置 CRUD**
   - POST `/api/v1/mcps` - 创建 MCP 配置
   - GET `/api/v1/mcps` - 分页查询 MCP 列表
   - GET `/api/v1/mcps/:uuid` - 获取单条 MCP 配置
   - POST `/api/v1/mcps/:uuid/delete` - 删除 MCP 配置

2. **MCP 连接池管理**
   - 支持 stdio/http/sse 三种连接类型
   - stdio 类型：进程生命周期管理
   - http/sse 类型：HTTP 连接复用和自动重连

3. **MCP 连接测试**
   - POST `/api/v1/mcps/:uuid/test` - 测试 MCP 连接
   - 返回连接状态和可用工具列表

### 功能范围（P1）- 本期不做

- MCP 工具调用（POST `/api/v1/mcps/:uuid/call`）- 预留接口

## 技术方案

### 方案选型

- **技术选型**：Go + Gin + GORM，与现有项目保持一致
- **设计模式**：DDD 分层架构（Repository + UseCase + Handler），与 knowledge 模块一致
- **MCP 协议**：参考 Model Context Protocol 规范，实现 stdio 进程通信和 http/sse 连接管理

### 核心实现步骤

1. **创建 DTO 层**（`internal/interfaces/http/dto/mcp_dto.go`）
   - CreateMCPRequest / CreateMCPResponse
   - ListMCPRequest / ListMCPResponse
   - GetMCPRequest / GetMCPResponse
   - DeleteMCPRequest / DeleteMCPResponse
   - TestMCPRequest / TestMCPResponse

2. **创建 Repository 层**（`internal/db/repository/mcp_repo.go`）
   - MCPDbRepository 接口定义
   - 实现 Create / GetByUUID / List / Delete 方法

3. **创建 UseCase 层**（`internal/application/usecase/mcp/`）
   - create_mcp.go - 创建 MCP 配置
   - list_mcp.go - 分页查询 MCP 列表
   - get_mcp.go - 获取单条 MCP 配置
   - delete_mcp.go - 删除 MCP 配置（含软删除）
   - test_mcp.go - 测试 MCP 连接

4. **创建 Handler 层**（`internal/interfaces/http/handler/mcp_handler.go`）
   - 聚合所有 UseCase
   - 实现 HTTP Handler 方法

5. **创建 MCP 连接池**（`internal/mcp/`）
   - mcp/client.go - 连接池管理器
   - mcp/stdio.go - stdio 进程管理
   - mcp/http.go - http/sse 连接管理

6. **路由注册**（`internal/interfaces/http/router.go`）
   - 添加 MCP 路由组

7. **Wire 依赖注入**（`internal/bootstrap/wire.go`）
   - 注册 MCP 相关 Provider

8. **更新配置**（`internal/infrastructure/config/config.go`）
   - 添加 MCP 连接池配置参数

### MCP 连接池设计

```go
// MCPClient 连接池接口
type MCPClient interface {
    // Connect 建立连接
    Connect(ctx context.Context, config *models.MCPConfig) error
    // Disconnect 断开连接
    Disconnect(ctx context.Context, uuid string) error
    // TestConnection 测试连接，返回可用工具列表
    TestConnection(ctx context.Context, uuid string) ([]string, error)
    // GetStatus 获取连接状态
    GetStatus(uuid string) (string, error)
}

// 连接池结构
type ClientPool struct {
    sync.RWMutex
    clients map[string]MCPClient  // uuid -> client
}
```

### 三种连接类型处理

| 类型 | 连接建立 | 断开策略 | 测试方式 |
|------|----------|----------|----------|
| stdio | 启动子进程 | Kill 进程 | 发送 initialize + list_tools |
| http | HTTP POST | 保持连接池 | HTTP POST initialize |
| sse | SSE 连接 | 关闭 EventSource | SSE 握手 |

## 接口定义

| 接口 | 方法 | 路径 | 请求参数 | 响应 |
|------|------|------|----------|------|
| 创建MCP | POST | /api/v1/mcps | `{name, description, type, command, args, env, url, headers}` | `{uuid, name, type, status}` |
| 列表MCP | GET | /api/v1/mcps | `page, page_size, type, status` | `{total, items[...]}` |
| 获取MCP | GET | /api/v1/mcps/:uuid | - | `{uuid, name, type, config, status, last_error}` |
| 删除MCP | POST | /api/v1/mcps/:uuid/delete | - | `{success: true}` |
| 测试MCP | POST | /api/v1/mcps/:uuid/test | `{timeout}` | `{status, tools: [...], elapsed_ms}` |

### 请求/响应结构

```go
// CreateMCPRequest
type CreateMCPRequest struct {
    Name        string   `json:”name” binding:”required”`
    Description string   `json:”description”`
    Type        string   `json:”type” binding:”required,oneof=stdio http sse”`
    Command     string   `json:”command”`      // stdio 类型
    Args        []string `json:”args”`         // stdio 类型
    Env         []string `json:”env”`          // stdio 类型
    URL         string   `json:”url”`          // http/sse 类型
    Headers     map[string]string `json:”headers”` // http/sse 类型
}

// CreateMCPResponse
type CreateMCPResponse struct {
    Code    int              `json:”code”`
    Message string           `json:”message”`
    Data    CreateMCPData    `json:”data”`
    Timestamp time.Time      `json:”timestamp”`
}

type CreateMCPData struct {
    UUID     string `json:”uuid”`
    Name     string `json:”name”`
    Type     string `json:”type”`
    Status   string `json:”status”`
}

// TestMCPResponse
type TestMCPResponse struct {
    Code    int              `json:”code”`
    Message string           `json:”message”`
    Data    TestMCPData      `json:”data”`
    Timestamp time.Time      `json:”timestamp”`
}

type TestMCPData struct {
    Status     string   `json:”status”`
    Tools      []string `json:”tools”`
    ElapsedMs  int64    `json:”elapsed_ms”`
    Error      string   `json:”error,omitempty”`
}
```
