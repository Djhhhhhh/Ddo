# MCP 管理 API - 技术方案

## 背景与目标

实现 MCP (Model Context Protocol) 服务器的管理功能，支持配置管理、连接池维护和测试接口。MCP 是 AI 应用与外部工具交互的协议，server-go 作为网关需要管理这些连接配置，并提供给 LLM 代理使用。

支持的 MCP 类型：
- **stdio**: 本地命令行程序，通过标准输入输出通信
- **http**: HTTP 接口服务
- **sse**: Server-Sent Events 实时推送服务

## 需求概述

1. **MCP 配置 CRUD**：增删改查 MCP 服务器配置
2. **连接池管理**：stdio 进程管理、http/sse 连接复用
3. **MCP 测试**：测试连接状态，返回可用工具列表
4. **工具调用代理**：代理 LLM 的 MCP 工具调用请求

## 技术方案

### 方案选型

| 功能 | 技术选型 | 说明 |
|------|----------|------|
| 配置存储 | MySQL | MCPConfig 模型已定义 |
| stdio 进程 | os/exec | Go 标准库进程管理 |
| HTTP 客户端 | net/http | 原生 HTTP 客户端 |
| SSE 客户端 | eventsource | 或自研 SSE 解析 |
| MCP 协议 | JSON-RPC | MCP 基于 JSON-RPC 2.0 |

### 核心实现步骤

1. **MCP Repository** (`internal/db/repository/mcp_repo.go`)
   - MCPConfig CRUD 操作
   - 按类型/状态查询

2. **MCP 客户端实现** (`internal/mcp/`)
   - `client.go`: 通用 MCP 客户端接口
   - `stdio_client.go`: stdio 类型实现（进程管理）
   - `http_client.go`: http 类型实现（HTTP 请求）
   - `sse_client.go`: sse 类型实现（SSE 连接）
   - 统一接口：Connect, Disconnect, Ping, ListTools, CallTool

3. **MCP 连接池** (`internal/mcp/pool.go`)
   - 管理活跃连接
   - stdio 类型：进程生命周期管理
   - http/sse 类型：连接复用和健康检查
   - 自动重连机制

4. **UseCase 层** (`internal/application/usecase/mcp/...`)
   - CreateMCPUseCase: 创建配置，验证参数
   - ListMCPUseCase: 查询 MCP 列表
   - GetMCPUseCase: 获取配置详情
   - UpdateMCPUseCase: 更新配置
   - DeleteMCPUseCase: 软删除配置
   - TestMCPUseCase: 测试连接，返回可用工具
   - CallMCPUseCase: 代理工具调用

5. **HTTP Handler** (`internal/interfaces/http/handler/mcp_handler.go`)
   - RESTful API 端点实现

## 接口定义

### MCP 配置管理

| 接口 | 方法 | 路径 | 功能 |
|------|------|------|------|
| 创建配置 | POST | `/api/v1/mcps` | 创建 MCP 配置 |
| 查询列表 | GET | `/api/v1/mcps` | 分页查询，支持类型筛选 |
| 获取单条 | GET | `/api/v1/mcps/:uuid` | 配置详情和连接状态 |
| 更新配置 | POST | `/api/v1/mcps/:uuid/update` | 更新配置，自动重连 |
| 删除配置 | POST | `/api/v1/mcps/:uuid/delete` | 软删除，关闭连接 |

### MCP 测试与调用

| 接口 | 方法 | 路径 | 功能 |
|------|------|------|------|
| 测试连接 | POST | `/api/v1/mcps/:uuid/test` | 测试连接，返回工具列表 |
| 工具调用 | POST | `/api/v1/mcps/:uuid/call` | 代理工具调用 |

### 请求/响应示例

**创建 stdio 类型 MCP - 请求**
```json
POST /api/v1/mcps
{
  "name": "filesystem",
  "description": "本地文件系统访问",
  "type": "stdio",
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-filesystem"],
  "env": {
    "HOME": "/Users/username"
  }
}
```

**创建 http 类型 MCP - 请求**
```json
POST /api/v1/mcps
{
  "name": "weather-api",
  "description": "天气查询服务",
  "type": "http",
  "url": "http://localhost:3001/mcp",
  "headers": {
    "Authorization": "Bearer token"
  }
}
```

**工具调用 - 请求**

## MCP 协议简介

MCP 使用 JSON-RPC 2.0 协议通信：

**初始化请求**
```json
{
  "jsonrpc": "2.0",
  "method": "initialize",
  "params": {
    "protocolVersion": "2024-11-05",
    "capabilities": {},
    "clientInfo": { "name": "ddo-server", "version": "1.0.0" }
  },
  "id": 1
}
```

**工具列表请求**
```json
{ "jsonrpc": "2.0", "method": "tools/list", "id": 2 }
```

**工具调用请求**
```json
{
  "jsonrpc": "2.0",
  "method": "tools/call",
  "params": {
    "name": "read_file",
    "arguments": { "path": "/tmp/test.txt" }
  },
  "id": 3
}
```

## 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| stdio 进程启动失败 | 更新 MCP 状态为 error，记录 last_error |
| http 连接超时 | 返回错误，不更新状态（临时故障）|
| SSE 连接断开 | 自动重连机制，3次失败后标记为 error |
| 工具调用失败 | 返回 Tool 执行错误，父级不报错 |
| 配置验证失败 | 创建/更新时返回 400 错误 |
