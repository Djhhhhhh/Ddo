# server-go MySQL 连接 - 技术方案

## 背景与目标

server-go 作为 Ddo 平台的核心网关服务，需要持久化存储知识库元数据、定时任务配置、MCP 配置以及任务执行日志等数据。本任务实现基于 GORM 的 MySQL 连接管理，为后续业务模块提供数据库访问能力。

## 需求概述

1. **数据库连接管理**: 实现 MySQL 8.0 连接池，支持配置文件和环境变量读取 DSN
2. **GORM 集成**: 使用 GORM v2 作为 ORM 框架，支持自动迁移和连接池管理
3. **优雅关闭**: 服务关闭时优雅释放数据库连接
4. **健康检查**: 在 `/health` 端点中集成 MySQL 连接状态检查

## 技术方案

### 方案选型

- **ORM 框架**: GORM v2（Go 生态最成熟的 ORM，支持 MySQL、PostgreSQL、SQLite）
- **MySQL 驱动**: `go-sql-driver/mysql`（GORM 推荐的官方驱动）
- **连接池配置**: GORM 内置连接池，支持最大连接数、空闲连接数、连接生命周期配置
- **配置读取**: 优先从环境变量读取，其次从配置文件 `~/.ddo/config.yaml` 读取

### 核心实现步骤

1. **初始化数据库模块结构**
   - 创建 `internal/db/mysql.go` 实现 GORM 连接初始化
   - 创建 `internal/db/models/` 目录存放实体模型定义

2. **配置 DSN 读取**
   - 实现从配置文件读取 MySQL DSN
   - 支持环境变量覆盖（`DDO_MYSQL_DSN`）
   - 默认 DSN: `root:root@tcp(127.0.0.1:3306)/ddo?charset=utf8mb4&parseTime=True&loc=Local`

3. **GORM 连接初始化**
   - 使用 `gorm.Open(mysql.Open(dsn), config)` 创建连接
   - 配置连接池: `SetMaxOpenConns(100)`, `SetMaxIdleConns(10)`, `SetConnMaxLifetime(time.Hour)`
   - 启用日志: Debug 模式输出 SQL，Release 模式静默

4. **数据库模型定义**
   - `Knowledge` - 知识库元数据表
   - `Timer` - 定时任务配置表
   - `TimerLog` - 定时任务执行日志表
   - `MCPConfig` - MCP 配置表
   - 所有表使用 `uuid` 作为主键，`created_at`/`updated_at` 作为时间戳

5. **自动迁移实现**
   - 服务启动时调用 `db.AutoMigrate(&models...)` 自动创建/更新表结构
   - 避免数据丢失，仅执行安全的 schema 变更

6. **Wire 依赖注入集成**
   - 在 `wire.go` 中添加 `db.NewMySQLConn` Provider
   - 将 `*gorm.DB` 注入到需要使用数据库的 Service 中

7. **优雅关闭支持**
   - 实现 `db.Close()` 方法，返回 `sqlDB.Close()` 的 cleanup 函数
   - 在 `main.go` 中捕获 SIGTERM/SIGINT 信号，调用 cleanup

8. **健康检查集成**
   - 在 `/health` 端点中调用 `sqlDB.Ping()` 检查连接
   - 返回 MySQL 连接状态: `{"mysql": "connected"}` 或 `{"mysql": "disconnected"}`

## 接口定义（内部使用）

| 接口/函数 | 路径 | 说明 |
|-----------|------|------|
| `NewMySQLConn(cfg *config.Config) (*gorm.DB, error)` | internal/db/mysql.go | 创建 GORM 连接 |
| `db.Close() error` | internal/db/mysql.go | 关闭数据库连接 |
| `AutoMigrate(models ...interface{}) error` | GORM 内置 | 自动迁移表结构 |

## 目录结构

```
server-go/
├── internal/
│   ├── db/
│   │   ├── mysql.go          # GORM 连接初始化
│   │   └── models/
│   │       ├── knowledge.go  # 知识库模型
│   │       ├── timer.go      # 定时任务模型
│   │       ├── timer_log.go  # 任务日志模型
│   │       └── mcp.go        # MCP 配置模型
│   └── config/
│       └── config.go         # 配置结构体添加 MySQL DSN 字段
├── cmd/server/
│   └── main.go               # 调用 db.Close() 优雅关闭
├── internal/api/handler/
│   └── health.go             # 集成 MySQL ping 检查
└── wire.go                   # 添加 db.NewMySQLConn Provider
```
