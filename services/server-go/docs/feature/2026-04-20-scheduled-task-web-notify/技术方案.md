# 定时任务触发 Web 灵动岛通知 - 技术方案

## 背景与目标

定时任务执行时需要触发 web 端的灵动岛进行通知，让用户实时了解任务执行状态。

## 需求概述

1. 定时任务执行时发送通知到 web 端
2. 复用已有的灵动岛组件展示通知
3. 支持不同任务类型的差异化通知

## 技术方案

### 方案选型

- **技术选型**: Go HTTP API + Electron 轮询机制
- **设计模式**: 生产者-消费者模式（服务端生产通知，Electron 消费）
- **通信方式**: Electron 每 10 秒轮询 `/api/notifications/subscribe` 获取新通知

### 调用链路

```
Go 定时任务执行
    ↓
调用 NotificationService.AddNotification()
    ↓
通知存储在内存队列（带过期时间）
    ↓
Electron 轮询 GET /api/notifications/subscribe
    ↓
返回待显示的通知列表
    ↓
Electron notification.ts showNotification()
    ↓
根据 level 调用 showDingIslandNotification()
    ↓
灵动岛窗口显示通知
```

### 核心实现步骤

1. **服务端实现**
   - 创建 NotificationService 管理通知队列
   - 实现 `GET /api/notifications/subscribe` 接口
   - 在定时任务执行点调用 `AddNotification()`

2. **通知数据结构**

```go
type Notification struct {
    ID        string    `json:"id"`
    Title     string    `json:"title"`
    Body      string    `json:"body"`
    Level     string    `json:"level"` // normal | important | urgent
    Timestamp int64     `json:"timestamp"`
    Type      string    `json:"type"`    // scheduled_task
    TaskName  string    `json:"taskName"`
    Status    string    `json:"status"`  // started | completed | failed
}
```

3. **定时任务集成**
   - 在任务开始时发送 `started` 通知（level: normal）
   - 在任务完成时发送 `completed` 通知（level: important）
   - 在任务失败时发送 `failed` 通知（level: urgent）

## 接口定义

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 订阅通知 | GET | /api/notifications/subscribe | 获取待显示的通知列表 |

**响应格式**:
```json
{
  "notifications": [
    {
      "id": "task-123",
      "title": "数据同步任务",
      "body": "任务执行成功，同步 100 条记录",
      "level": "important",
      "timestamp": 1716192000000,
      "type": "scheduled_task",
      "taskName": "数据同步",
      "status": "completed"
    }
  ]
}
```
