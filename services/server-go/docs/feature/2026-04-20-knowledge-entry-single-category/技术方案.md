# 知识库词条单类别解析 - 技术方案

## 背景与目标

当前创建知识库词条时，对于一个词条会解析出多个种类，导致数据库中显示多个重复词条。需要修改为每次只解析出一个大类，避免数据冗余。

## 需求概述

1. 修改词条解析逻辑，确保只返回一个主要类别
2. 当有多个候选类别时，选择最相关的一个
3. 保持原有 API 接口不变，仅修改内部处理逻辑

## 技术方案

### 方案选型

- **技术选型**: Go + 现有的知识库解析模块
- **设计模式**: 优先级评分 + 单一选择
- **修改范围**: 服务端解析逻辑

### 核心实现步骤

1. **定位解析逻辑**
   - 查找词条创建相关的 handler 和 service
   - 定位类别解析的具体代码位置

2. **修改解析策略**
   - 将返回多个类别的逻辑改为返回单个类别
   - 添加优先级判断：选择置信度最高的类别
   - 如置信度相同，选择第一个或按预设规则选择

3. **数据层调整**
   - 检查数据库 schema，确保单类别存储兼容
   - 如有必要，添加类别优先级字段

4. **测试验证**
   - 验证不同输入只返回一个类别
   - 验证边界情况（无类别、多个相同置信度类别）

## 接口定义

接口保持不变，仅修改内部处理逻辑：

| 接口 | 方法 | 路径 | 请求参数 | 响应 |
|------|------|------|----------|------|
| 创建词条 | POST | /api/knowledge/entries | `{title, content, ...}` | `{id, category, ...}` |

**响应变更**: `category` 字段从 `[]string` 改为 `string` 类型
