# 定时任务 API (Timer API) - 技术方案

## 背景与目标

实现定时任务调度系统，支持 Cron 表达式配置、HTTP 回调触发和执行日志记录。server-go 负责定时任务的 CRUD 管理、Cron 调度和回调执行，使用 robfig/cron 库解析 Cron 表达式，BadgerDB 队列作为消息中间件。

## 需求概述

1. **定时任务 CRUD**：增删改查定时任务配置
2. **Cron 调度**：基于 robfig/cron 实现精确调度
3. **HTTP 回调**：任务触发时执行配置的 HTTP 请求
4. **执行日志**：记录每次任务执行的状态、输出、错误和耗时
5. **任务控制**：支持暂停、恢复、手动触发

## 技术方案

### 方案选型

| 组件 | 选型 | 说明 |
|------|------|------|
| Cron 解析 | robfig/cron/v3 | 标准 Cron 表达式，支持时区 |
| 调度器 | 自研 Scheduler | 基于 cron.Cron 封装 |
| 任务执行 | HTTP Client | 执行配置的回调请求 |
| 日志存储 | MySQL | TimerLog 模型已定义 |
| 消息队列 | BadgerDB | 已完成 p2-17，用于异步执行 |

### 核心实现步骤

1. **增强 Scheduler** (`internal/scheduler/scheduler.go`)
   - 集成 robfig/cron 替代自研调度循环
   - 支持动态添加/删除 Cron 任务
   - 时区支持（默认 Local，可配置）
   - 启动时从数据库加载 active 任务

2. **定时任务 Repository** (`internal/db/repository/timer_repo.go`)
   - Timer CRUD 操作
   - TimerLog 查询和写入
   - 按状态查询任务列表

3. **UseCase 层** (`internal/application/usecase/timer/...`)
   - CreateTimerUseCase: 创建定时任务，解析 Cron 验证
   - ListTimerUseCase: 分页查询任务列表
   - GetTimerUseCase: 获取任务详情和执行统计
   - UpdateTimerUseCase: 更新任务配置
   - PauseTimerUseCase: 暂停任务（cron.Stop + 状态更新）
   - ResumeTimerUseCase: 恢复任务（cron.Start + 状态更新）
   - DeleteTimerUseCase: 软删除任务
   - ListTimerLogsUseCase: 查询执行日志

4. **回调执行器** (`internal/application/service/callback_executor.go`)
   - 从队列消费 Timer 消息
   - 执行 HTTP 回调（GET/POST）
   - 记录执行结果到 MySQL
   - 支持 Headers 和 Body 模板

5. **HTTP Handler** (`internal/interfaces/http/handler/timer_handler.go`)
   - RESTful API 端点实现
   - 请求验证和响应封装

## 接口定义

### 定时任务管理

| 接口 | 方法 | 路径 | 功能 |
|------|------|------|------|
| 创建任务 | POST | `/api/v1/timers` | 创建定时任务，自动加入调度器 |
| 查询列表 | GET | `/api/v1/timers` | 分页查询，支持状态筛选 |
| 获取单条 | GET | `/api/v1/timers/:uuid` | 任务详情和统计信息 |
| 更新任务 | POST | `/api/v1/timers/:uuid/update` | 更新配置，重新调度 |
| 暂停任务 | POST | `/api/v1/timers/:uuid/pause` | 暂停调度 |
| 恢复任务 | POST | `/api/v1/timers/:uuid/resume` | 恢复调度 |
| 删除任务 | POST | `/api/v1/timers/:uuid/delete` | 软删除任务 |
| 手动触发 | POST | `/api/v1/timers/:uuid/trigger` | 立即执行一次 |

### 执行日志

| 接口 | 方法 | 路径 | 功能 |
|------|------|------|------|
| 查询日志 | GET | `/api/v1/timers/:uuid/logs` | 分页查询执行历史 |

## 关键实现细节

**Cron 表达式验证**
```go
func ValidateCron(expr string) error {
    _, err := cron.ParseStandard(expr)
    return err
}
```

**调度器集成 robfig/cron**
```go
type Scheduler struct {
    cron   *cron.Cron
    queue  queue.Queue
    logger *zap.Logger
}

func (s *Scheduler) AddJob(timer *models.Timer) error {
    entryID, err := s.cron.AddFunc(timer.CronExpr, func() {
        s.triggerJob(timer)
    })
    // 保存 entryID 用于后续移除
}
```

**回调执行器消费队列**
```go
func (e *CallbackExecutor) consumeTimerMessages(ctx context.Context) {
    e.queue.Subscribe(ctx, "timer", func(msg queue.Message) error {
        var payload TimerPayload
        json.Unmarshal(msg.Payload, &payload)
        
        start := time.Now()
        resp, err := e.executeHTTP(payload)
        duration := time.Since(start).Milliseconds()
        
        // 记录日志
        e.logRepo.Create(&models.TimerLog{
            TimerUUID: payload.TimerUUID,
            Status:    status(err),
            Duration:  duration,
            Output:    string(resp),
            Error:     errMsg(err),
        })
        
        return nil
    })
}
```
