# RAG意图路由 - 交互逻辑详细示例

> 归属：RAG意图路由重构技术方案
> 用途：指导CLI交互逻辑重构

## 概述

本文档详细展示CLI层的交互流程，包含完整的状态切换和UI展示逻辑。

---

## 核心交互原则

1. **每步有反馈**：用户知道当前在处理哪个阶段
2. **状态可见性**：显示中间状态（分析中/检索中/生成中）
3. **渐进式呈现**：先显示意图，再显示结果
4. **清晰的数据来源**：如果是RAG结果，显示来源引用

---

## 场景1：知识查询（完整流程）

### 用户输入
```
ddo ~ 怎么配置定时任务每小时执行一次？
```

### 阶段1：意图分析

**CLI状态机**:
```typescript
state.status = 'analyzing'
showSpinner('🔍 分析意图...')
```

**终端显示**:
```
🔍 分析意图...
```

**API调用**:
```typescript
// 向Go服务发送请求
const response = await api.conversationChat({
    query: "怎么配置定时任务每小时执行一次？",
    stream: true
})

// 监听SSE事件
response.on('event', (event) => {
    if (event.type === 'intent_detected') {
        handleIntentDetected(event.data)
    }
})
```

## 交互状态定义

```typescript
// 会话状态
interface ConversationState {
    // 当前阶段
    status: 'idle' | 'analyzing' | 'clarifying' | 'retrieving' | 'generating' | 'completed' | 'error'
    
    // 意图信息
    intent?: {
        type: string
        needKnowledge: boolean
        confidence: number
    }
    
    // 检索结果
    retrievedDocs?: {
        id: string
        score: number
        preview: string
    }[]
    
    // 生成内容
    answerBuffer: string
    isStreaming: boolean
    
    // 错误信息
    error?: {
        type: string
        message: string
        isRetryable: boolean
    }
}

// 状态配置（用于UI渲染）
const statusConfig = {
    analyzing: {
        icon: '🔍',
        text: '分析意图...',
        color: chalk.cyan,
    },
    clarifying: {
        icon: '❓',
        text: '需要确认...',
        color: chalk.yellow,
    },
    retrieving: {
        icon: '📚',
        text: '检索知识库...',
        color: chalk.magenta,
    },
    generating: {
        icon: '✨',
        text: '生成回答...',
        color: chalk.green,
    },
    completed: {
        icon: '✓',
        text: '完成',
        color: chalk.green,
    },
    error: {
        icon: '⚠️',
        text: '出错',
        color: chalk.red,
    },
}
```

## 调用关系图

```
repl/index.ts
    │
    ├── 用户输入
    │
    ▼
ConversationHandler.handle(input)
    │
    ├── showStatus('analyzing')
    │
    ▼
api.conversationChat({ query: input, stream: true })
    │
    ▼
server-go: POST /api/v1/conversation/chat
    │
    ├── 意图识别
    ├── 路由决策
    └── RAG/Chat/Tool
    │
    ▼
SSE Stream Events
    │
    ├── event: intent_detected
    │   └── handler.showIntent(intent)
    │
    ├── event: retrieving
    │   └── handler.showRetrieving()
    │
    ├── event: docs_found
    │   └── handler.showDocs(docs)
    │
    ├── event: delta (流式)
    │   └── handler.streamChunk(content)
    │
    └── event: completed
        └── handler.showResult()
```
