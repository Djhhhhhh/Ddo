# Bugfix - CLI Bug 修复

## Bug 1: 知识库优先检索模式

### 问题描述
- CLI 执行时默认直接处理用户输入，不会自动检索知识库
- 用户无法快速切换是否优先从知识库获取数据

### 技术方案

**修改文件**: `services/cli/src/repl/index.ts`, `services/cli/src/repl/mode.ts`, `services/cli/src/repl/commands/index.ts`

#### 1. 添加知识库模式状态
```typescript
// services/cli/src/repl/mode.ts
export enum ReplMode {
  Default = 'default',
  Chat = 'chat',
  Kb = 'kb',
  Timer = 'timer',
  Mcp = 'mcp',
}

// 添加模式管理器扩展
interface ModeManager {
  // ... 现有接口
  /** 知识库优先模式 */
  kbPriorityMode: boolean;
  /** 切换知识库优先模式 */
  toggleKbPriority: () => void;
  /** 获取提示符后缀 */
  getPromptSuffix: () => string;
}
```

#### 2. 在用户最后一条消息下方添加紫色标识
```typescript
// services/cli/src/repl/commands/index.ts
// 在显示用户输入后，添加知识库优先模式标识
function displayUserInput(fullText: string, kbPriorityMode: boolean): void {
  console.log(chalk.cyan('你:'), fullText);

  // 如果开启了知识库优先模式，显示紫色标识
  if (kbPriorityMode) {
    console.log(chalk.gray('   └─ '), chalk.magenta('📚 知识库优先'));
  }
}
```

#### 3. Tab 键监听切换
```typescript
// services/cli/src/repl/index.ts
// 在 rl.on('line') 之前添加 Tab 键监听
rl.on('keypress', (str, key) => {
  if (key.name === 'tab') {
    modeManager.toggleKbPriority();
    rl.write('\r\n');  // 换行
    console.log(chalk.magenta(`📚 知识库优先模式: ${modeManager.kbPriorityMode ? '开启' : '关闭'}`));
    rl.prompt();
  }
});
```

#### 4. 知识库优先模式下的处理逻辑
```typescript
// services/cli/src/repl/commands/index.ts - handleUnknownCommand
async function handleUnknownCommand(name: string, ctx: CommandContext): Promise<boolean> {
  const fullText = [name, ...ctx.args].join(' ').trim();

  // 如果开启了知识库优先模式，先检索知识库
  if (ctx.modeManager.kbPriorityMode && fullText) {
    try {
      const { getApiClient } = await import('../../services/api-client');
      const apiClient = getApiClient();

      // 调用知识库问答
      const kbResult = await apiClient.askKnowledge(fullText);
      if (kbResult && kbResult.content) {
        console.log(chalk.magenta('📚 知识库回复:'), kbResult.content);
        console.log();
        return true;
      }
    } catch (err) {
      // 知识库检索失败，继续正常的意图识别流程
      console.log(chalk.gray('知识库检索失败，继续分析意图...'));
    }
  }

  // ... 继续正常的 NLP 意图识别流程
}
```

## Bug 3: 新增知识库时 categories 不同步

### 问题描述
查看 `create_knowledge.go` 代码发现，categories 的自动创建逻辑**只在 AI 分析返回 `isNewCategories[i] == true`** 时才会创建。如果 AI 分析没有正确返回 `isNewCategories` 字段，或者该字段为空数组，则新分类不会被创建。

### 技术方案

**修改文件**: `services/server-go/internal/application/usecase/knowledge/create_knowledge.go`

#### 修复逻辑 (第 105 行)
```go
// 原逻辑：isNewCategories == nil || len(isNewCategories) <= i || isNewCategories[i]
// 新逻辑：isNewCategories == nil || len(isNewCategories) == 0 || i >= len(isNewCategories) || isNewCategories[i]
shouldCreate := isNewCategories == nil || len(isNewCategories) == 0 || i >= len(isNewCategories) || isNewCategories[i]
```

**修复点说明**：
- 增加 `len(isNewCategories) == 0` 判断，确保当 AI 返回空数组时也能正确处理
- 修改 `len(isNewCategories) <= i` 为 `i >= len(isNewCategories)`，逻辑相同但更清晰

#### 额外检查：用户提供的 category 也自动创建 (第 122-125 行)
```go
// 额外检查：如果用户提供了 category 但不在 categoryIDs 中，也应该自动创建
if input.Category != "" {
    categoryIDs = append(categoryIDs, input.Category)
}
```
