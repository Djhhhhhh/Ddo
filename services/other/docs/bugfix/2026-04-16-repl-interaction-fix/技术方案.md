# REPL 交互逻辑重构 - 技术方案

## 背景与目标

当前 REPL 存在以下问题：

1. **状态显示问题**：`/status` 显示的服务信息不准确，MySQL 不应作为独立服务显示
2. **模式感知问题**：用户能感知到"进入不同模式"（Timer/Kb/Mcp），体验割裂
3. **交互流程问题**：创建定时任务等操作参数获取流程不顺畅，用户不知道需要输入什么

**目标**：让用户只感知一个对话模式，所有操作通过自然对话 + 参数引导完成。

## 问题诊断

### 问题 1：状态显示不准确

**当前代码** (`status.ts:24-40`)：
```typescript
// 问题 1：显示 MySQL 作为独立服务
console.log(`    MySQL: ${h.mysql === 'connected' ? chalk.green(h.mysql) : chalk.red(h.mysql)}`);

// 问题 2：llm-py 状态未正确获取
const llmPyStatus = m.services?.llm_py;
console.log(`  ${llmPyStatus === 'running' ? chalk.green('●') : chalk.red('●')} llm-py   ${llmPyStatus ? chalk.green(llmPyStatus) : chalk.red('stopped')}`);
console.log(`  ${chalk.green('●')} MySQL    ${chalk.green('running')}`); // MySQL 不应该显示
```

**期望输出**：
```
服务状态:
────────────────────────────────────
  ● server-go  running
  ● llm-py     running
  ● CLI        running
```

### 问题 2：模式切换割裂体验

**当前流程**：
```
用户: 帮我创建一个每小时执行的任务
NLP: timer.create → switch_mode → Timer 模式
REPL: 显示 "已进入定时任务模式"，用户感知到模式切换
用户: 在 Timer 模式中输入 add "0 * * * *" ...
```

**问题**：
- `mode-switch.ts` 创建了 `/timer`、`/kb`、`/mcp` 命令，切换模式时显示 welcome message
- 用户明确感知到"进入了不同的子命令模式"
- `chat.ts` 中 `setMode(ReplMode.Chat)` 也会改变提示符 `ddo/chat> `

**期望流程**：
```
用户: 帮我创建一个每小时执行的任务
NLP: timer.create
REPL: 自动执行 add 命令，如果参数不足则引导用户输入
用户: http://localhost:8080/callback
REPL: 创建成功，返回结果
整个过程用户只在一个统一的对话界面中
```

### 问题 3：参数引导缺失

**当前代码** (`timer-commands.ts:85-92`)：
```typescript
if (!cron || !url) {
  console.log(chalk.yellow('用法: /timer add <cron表达式> <回调URL> [方法]'));
  console.log(chalk.gray('示例: /timer add "0 * * * *" "http://localhost:8080/callback" POST'));
  if (nlpParameters) {
    console.log(chalk.gray('提示: 检测到自然语言参数 - Cron:'), cron || '未识别', chalk.gray(', URL:'), url || '未识别');
  }
  return true;  // 直接返回，没有引导用户下一步
}
```

**期望行为**：
- 语义完整时直接执行：用户输入的语义如果包含足够的最少参数，直接调用 API 执行，不中断对话
- 语义不完整时引导输入：参数不足时通过提示框形式引导用户输入缺少的参数
- 渐进式交互：用户输入一个参数后，可继续补充其他参数，直到收集到所有必填参数才执行
- 对话不中断：整个交互过程保持在同一个对话流程中

```typescript
/**
 * 创建快捷命令（不切换模式，只是执行对应子命令）
 * 这些命令原本是模式切换命令，现在改为直接执行对应操作
 */
function createQuickCommand(
  name: string,
  subCommand: string,
  description: string
): ReplCommand {
  return {
    name,
    description,
    usage: `/${name}`,
    handler: async (ctx: CommandContext) => {
      // 直接执行对应子命令
      const cmd = registry.get(subCommand);
      if (cmd) {
        return await cmd.handler(ctx);
      }
      console.log(chalk.red(`命令 ${subCommand} 不存在`));
      return true;
    },
  };
}
```

### 变更 3：重构参数引导交互

#### Step 3.1：创建交互式参数收集器

**新增文件**: `services/cli/src/repl/commands/prompt-helper.ts`

```typescript
import readline from 'readline';
import chalk from 'chalk';

/**
 * 参数定义
 */
export interface ParameterDef {
  /** 参数名 */
  name: string;
  /** 显示名称 */
  label: string;
  /** 是否必填 */
  required: boolean;
  /** 默认值提示 */
  defaultHint?: string;
  /** 验证函数 */
  validate?: (value: string) => string | null;  // 返回 null 表示有效，错误消息表示无效
  /** 帮助说明 */
  help?: string;
}

/**
 * 交互式参数收集器
 */
export class InteractivePrompt {
  private rl: readline.Interface;

  constructor(rl: readline.Interface) {
    this.rl = rl;
  }

  /**
   * 提示用户输入单个参数
   */
  async promptParam(def: ParameterDef, currentValue?: string): Promise<string | null> {
    // 如果已有值，跳过
    if (currentValue) {
      console.log(chalk.gray(`  ${def.label}: ${currentValue} (已提供)`));
      return currentValue;
    }

    // 显示提示
    const requiredMark = def.required ? chalk.red('*') : chalk.gray('(可选)');
    const helpText = def.help ? chalk.gray(` - ${def.help}`) : '';
    const defaultText = def.defaultHint ? chalk.gray(` [${def.defaultHint}]`) : '';
    
    console.log();
    console.log(chalk.cyan(`${def.label}${requiredMark}${defaultText}${helpText}`));
    
    // 读取用户输入
    const answer = await this.promptUser();
    
    // 必填参数未提供
    if (!answer && def.required) {
      console.log(chalk.yellow(`  ${def.label} 是必填的，请输入`));
      return this.promptParam(def, currentValue);
    }
    
    // 验证输入
    if (answer && def.validate) {
      const error = def.validate(answer);
      if (error) {
        console.log(chalk.red(`  ${error}`));
        return this.promptParam(def, currentValue);
      }
    }
    
    return answer || null;
  }

  /**
   * 收集多个参数
   */
  async collectParams(
    paramDefs: ParameterDef[],
    initialValues?: Record<string, string>
  ): Promise<Record<string, string | null>> {
    const result: Record<string, string | null> = {};
    const initial = initialValues || {};

    for (const def of paramDefs) {
      const value = await this.promptParam(def, initial[def.name]);
      result[def.name] = value;
    }

    return result;
  }

  /**
   * 显示汇总确认
   */
  async confirmSummary(
    title: string,
    params: Record<string, string | null>
  ): Promise<boolean> {
    console.log();
    console.log(chalk.cyan(title));
    console.log(chalk.gray('─'.repeat(40)));
    
    for (const [key, value] of Object.entries(params)) {
      const displayValue = value || chalk.gray('(未设置)');
      console.log(`  ${chalk.yellow(key)}: ${displayValue}`);
    }
    
    console.log();
    const answer = await this.promptUserConfirm('确认创建? (y/n)');
    return answer;
  }

  /**
   * 提示用户输入
   */
  private promptUser(): Promise<string> {
    return new Promise((resolve) => {
      this.rl.question('', resolve);
    });
  }

  /**
   * 提示用户确认
   */
  private async promptUserConfirm(message: string): Promise<boolean> {
    const answer = await this.promptUser();
    return answer.toLowerCase() === 'y';
  }
}

/**
 * Cron 表达式验证
 */
export function validateCron(value: string): string | null {
  // 简单验证：5个字段用空格分隔
  const parts = value.trim().split(/\s+/);
  if (parts.length !== 5) {
    return 'Cron 表达式需要 5 个字段（分 时 日 月 周）';
  }
  return null;
}

/**
 * URL 验证
 */
export function validateUrl(value: string): string | null {
  try {
    new URL(value);
    return null;
  } catch {
    return '请输入有效的 URL';
  }
}
```

#### Step 3.2：重构 Timer Add 命令

**修改文件**: `services/cli/src/repl/commands/timer-commands.ts`

```typescript
import chalk from 'chalk';
import { ReplCommand, CommandContext } from './index';
import { ReplMode } from '../mode';
import { getApiClient } from '../../services/api-client';
import { InteractivePrompt, ParameterDef, validateCron, validateUrl } from './prompt-helper';

export const timerAddCommand: ReplCommand = {
  name: 'timer-add',  // 改名，避免与 mode-switch 中的 timer 命令冲突
  aliases: ['ta', 'timer:create'],
  description: '创建定时任务',
  usage: '/timer-add 或 自然语言: 帮我创建一个定时任务',
  handler: async (ctx) => {
    const { args, nlpParameters, rl } = ctx;
    const prompt = new InteractivePrompt(rl);

    // 定义需要收集的参数
    const paramDefs: ParameterDef[] = [
      {
        name: 'name',
        label: '任务名称',
        required: false,
        help: '给任务起个名字',
      },
      {
        name: 'cron',
        label: 'Cron 表达式',
        required: true,
        defaultHint: '0 * * * *',
        validate: validateCron,
        help: '分 时 日 月 周，例如：0 9 * * * 表示每天 9 点',
      },
      {
        name: 'url',
        label: '回调 URL',
        required: true,
        validate: validateUrl,
        help: '定时任务触发时访问的 URL',
      },
      {
        name: 'method',
        label: 'HTTP 方法',
        required: false,
        defaultHint: 'GET',
        help: 'GET/POST/PUT/DELETE',
      },
    ];

    // 从 NLP 参数和命令行参数中提取初始值
    const initialValues: Record<string, string> = {};
    
    if (args.length >= 1) initialValues.cron = args[0];
    if (args.length >= 2) initialValues.url = args[1];
    if (args.length >= 3) initialValues.method = args[2];
    
    if (nlpParameters) {
      if (!initialValues.cron && nlpParameters.cron) initialValues.cron = String(nlpParameters.cron);
      if (!initialValues.cron && nlpParameters.schedule) initialValues.cron = String(nlpParameters.schedule);
      if (!initialValues.url && nlpParameters.url) initialValues.url = String(nlpParameters.url);
      if (!initialValues.url && nlpParameters.callback_url) initialValues.url = String(nlpParameters.callback_url);
      if (!initialValues.method && nlpParameters.method) initialValues.method = String(nlpParameters.method);
      if (!initialValues.name && nlpParameters.name) initialValues.name = String(nlpParameters.name);
    }

    // 收集参数
    console.log(chalk.cyan('\n📋 创建定时任务'));
    console.log(chalk.gray('─'.repeat(40)));

    const collected = await prompt.collectParams(paramDefs, initialValues);

    // 确认汇总
    const name = collected.name || `Timer ${new Date().toISOString()}`;
    const confirmed = await prompt.confirmSummary('定时任务信息', {
      名称: name,
      Cron: collected.cron,
      回调URL: collected.url,
      方法: collected.method || 'GET',
    });

    if (!confirmed) {
      console.log(chalk.yellow('\n已取消创建'));
      return true;
    }

    // 调用 API 创建
    const apiClient = getApiClient();

    try {
      const result = await apiClient.createTimer({
        name,
        cron: collected.cron!,
        url: collected.url!,
        method: (collected.method as 'GET' | 'POST' | 'PUT' | 'DELETE') || 'GET',
      });

      console.log(chalk.green('\n✓ 定时任务创建成功!'));
      console.log(chalk.gray(`  UUID: ${result.uuid}`));
      console.log(chalk.gray(`  Cron: ${result.cron}`));
      console.log();
    } catch (err) {
      console.log(chalk.red('\n✗ 创建定时任务失败:'), err instanceof Error ? err.message : String(err));
    }

    return true;
  },
};
```

```typescript
    return true;
  },
};
```

### 变更 5：去除 NLP 和 Chat 的超时逻辑

**背景**：LLM 的推理时间是动态的，对于复杂问题可能需要较长的思考时间。设置固定超时时间会导致长思考任务被中断，造成不可预期的失败。

**修改文件 1**: `services/cli/src/services/nlp.ts`

去除请求超时控制：

```typescript
/**
 * 创建 NLP Service
 */
export function createNLPService(config: NLPServiceConfig) {
  const { baseUrl } = config;  // 移除 timeout

  /**
   * 发送 HTTP 请求（无超时限制）
   */
  async function request<T>(path: string, body: unknown): Promise<T> {
    const url = `${baseUrl}${path}`;

    logger.debug(`[nlp] 请求 ${url}`);

    const response = await fetch(url, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify(body),
      // 不设置 signal，由服务器端控制超时
    });

    if (!response.ok) {
      throw new NLPServiceError(`HTTP ${response.status}: ${response.statusText}`);
    }

    return await response.json() as T;
  }
  
  // ... 其他方法不变
}

export function getNLPService(): ReturnType<typeof createNLPService> {
  if (!defaultService) {
    defaultService = createNLPService({
      baseUrl: process.env.DDO_SERVER_GO_URL || 'http://localhost:8080',
      // 不再传递 timeout 参数
    });
  }
  return defaultService;
}
```

**修改文件 2**: `services/cli/src/services/api-client.ts`

去除 Chat 和通用请求的超时控制：

```typescript
export interface ApiClientConfig {
  serverGoUrl: string;
  // 移除 timeout: number;
}

/**
 * 创建 API Client
 */
export function createApiClient(config: ApiClientConfig) {
  const { serverGoUrl } = config;  // 移除 timeout

  /**
   * 发送 HTTP 请求（无超时限制）
   */
  async function request<T>(path: string, options?: RequestInit): Promise<T> {
    const url = `${serverGoUrl}${path}`;

    const response = await fetch(url, {
      ...options,
      headers: {
        'Content-Type': 'application/json',
        ...options?.headers,
      },
      // 不设置 signal
    });

    if (!response.ok) {
      const errorBody = await response.text();
      throw new Error(`API ${path} failed: ${response.status} ${errorBody}`);
    }

    const text = await response.text();
    if (!text) {
      return {} as T;
    }

    const json = JSON.parse(text);

    if (json && typeof json === 'object' && 'data' in json) {
      return json.data as T;
    }

    return json as T;
  }

  // Chat 方法不变（无超时）
  async function chat(messages: Message[], stream = false): Promise<ChatResponse> {
    return request<ChatResponse>('/api/v1/chat', {
      method: 'POST',
      body: JSON.stringify({ messages, stream }),
    });
  }

  return {
    // ...
  };
}

export function getApiClient(): ReturnType<typeof createApiClient> {
  if (!globalClient) {
    globalClient = createApiClient({
      serverGoUrl: process.env.DDO_SERVER_GO_URL || 'http://localhost:8080',
      // 不再传递 timeout
    });
  }
  return globalClient;
}
```

1. 移除 `NLPServiceConfig.timeout`
2. 移除 `ApiClientConfig.timeout`
3. 移除 fetch 请求中的 `AbortController` 和 `setTimeout`
4. 移除超时错误处理 `AbortError`
5. 健康检查的短超时（3秒/5秒）保持不变，因为它检查的是连接性，不是 LLM 推理
