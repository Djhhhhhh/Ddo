# LLM 对话记录存储与趋势统计 - 技术方案

## 背景与目标

Dashboard 需要展示 LLM 调用趋势，用户需要能够在本地查看历史对话记录并进行统计分析。同时需要为后续系统对话记忆系统预留扩展能力。

**核心目标**：
1. 在 llm-py 本地持久化存储所有 LLM 对话记录
2. 提供历史记录查询接口，支持分页和时间范围筛选
3. 提供趋势统计接口，支持按日/周/月聚合统计
4. 数据模型预留系统记忆功能的扩展字段

## 需求概述

### 功能需求

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 对话记录存储 | 每次 LLM 调用自动保存到本地 SQLite | P0 |
| 历史记录查询 | 支持分页、时间筛选、关键词搜索 | P0 |
| 趋势统计 API | 按时间维度统计调用次数、token 消耗 | P0 |
| 模型使用统计 | 按模型维度统计使用频率 | P1 |
| 记忆预留字段 | 预留 session_memory, context_summary 字段 | P1 |

### 非功能需求

- **存储位置**：`~/.ddo/data/llm/conversations.db`
- ** retention**：默认保留 90 天，可配置
- **性能**：查询响应 < 100ms（本地 SQLite）
- **并发**：支持高并发写入（SQLite WAL 模式）

## 技术方案

### 方案选型

| 维度 | 选型 | 理由 |
|------|------|------|
| 存储引擎 | SQLite | 零配置、本地文件、Python 原生支持 |
| ORM | SQLAlchemy | 成熟稳定、迁移支持、与 FastAPI 生态兼容 |
| 迁移工具 | Alembic | 标准迁移方案、支持版本管理 |
| 架构模式 | 异步 ORM | 与 FastAPI async 兼容 |

### 数据模型设计

#### 1. Conversation 表（对话会话）

```python
class Conversation(Base):
    """对话会话表 - 支持多轮对话分组"""
    __tablename__ = "conversations"
    
    id = Column(String(36), primary_key=True)  # UUID
    session_id = Column(String(36), index=True)  # 前端会话标识
    title = Column(String(255), nullable=True)  # 对话标题（自动摘要或用户设置）
    
    # 系统记忆预留字段
    context_summary = Column(Text, nullable=True)  # 上下文摘要（用于记忆系统）
    memory_enabled = Column(Boolean, default=False)  # 是否启用记忆增强
    memory_checkpoint = Column(String(36), nullable=True)  # 记忆检查点ID
    
    # 元数据
    source = Column(String(50), default="api")  # 来源：api, cli, web
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
    
    # 关联
    messages = relationship("Message", back_populates="conversation", cascade="all, delete-orphan")
```

#### 2. Message 表（消息记录）

```python
class Message(Base):
    """消息记录表 - 存储每次 LLM 交互"""
    __tablename__ = "messages"
    
    id = Column(String(36), primary_key=True)  # UUID
    conversation_id = Column(String(36), ForeignKey("conversations.id"), index=True)
    
    # 消息内容
    role = Column(String(20), nullable=False)  # system, user, assistant
    content = Column(Text, nullable=False)
    
    # LLM 调用信息
    model = Column(String(100), nullable=True)  # 使用的模型
    provider = Column(String(50), nullable=True)  # 提供商：openrouter, local
    
    # Token 统计（从 LLM 响应获取）
    prompt_tokens = Column(Integer, default=0)
    completion_tokens = Column(Integer, default=0)
    total_tokens = Column(Integer, default=0)
    
    # 性能指标
    latency_ms = Column(Integer, nullable=True)  # 响应延迟（毫秒）
    
    # 记忆预留字段
    embedding_id = Column(String(36), nullable=True)  # 向量存储ID（用于记忆检索）
    importance_score = Column(Float, nullable=True)  # 重要性评分（记忆优先级）
    
    # 元数据
    metadata = Column(JSON, default=dict)  # 扩展字段：intent, sources, 等
    created_at = Column(DateTime, default=datetime.utcnow)
    
    # 关联
    conversation = relationship("Conversation", back_populates="messages")
```

#### 3. ConversationStats 视图/表（统计聚合）

```python
class DailyStats(Base):
    """每日统计表 - 预聚合提升查询性能"""
    __tablename__ = "daily_stats"
    
    id = Column(String(36), primary_key=True)
    date = Column(Date, unique=True, index=True)  # 统计日期
    
    # 调用统计
    total_requests = Column(Integer, default=0)  # 总请求数
    total_tokens = Column(Integer, default=0)  # 总 token 消耗
    avg_latency_ms = Column(Integer, default=0)  # 平均延迟
    
    # 模型分布（JSON 存储）
    model_distribution = Column(JSON, default=dict)  # {"anthropic/claude-3.5": 100, ...}
    
    # 来源分布
    source_distribution = Column(JSON, default=dict)  # {"api": 50, "cli": 30, "web": 20}
    
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
```

### 核心实现步骤

#### Step 1: 数据库基础设施（llm-py）

1. **依赖安装**
   ```bash
   pip install sqlalchemy alembic aiosqlite
   ```

2. **数据库配置**
   - 配置文件：`app/core/config.py` 添加数据库配置
   - 默认路径：`~/.ddo/data/llm/conversations.db`
   - 连接池配置：WAL 模式、超时设置

3. **模型定义**
   - 创建 `app/db/models.py` 定义上述模型
   - 使用 SQLAlchemy 2.0 声明式风格

4. **迁移管理**
   - 初始化 Alembic
   - 创建初始迁移脚本

#### Step 2: 对话记录存储（llm-py）

1. **存储服务层**
   - 创建 `app/services/conversation_service.py`
   - 提供异步 CRUD 接口
   - 自动会话管理（自动创建 conversation）

2. **Chat API 集成**
   - 修改 `app/api/chat.py`
   - 在 completions/stream 接口中调用存储服务
   - 记录 token 使用、延迟等指标

3. **生命周期管理**
   - 在 `app/core/lifespan.py` 中初始化数据库连接

#### Step 3: 统计与查询 API（llm-py）

1. **查询接口**
   ```python
   GET /api/conversations          # 会话列表
   GET /api/conversations/{id}     # 会话详情（含消息）
   GET /api/messages               # 消息列表（支持筛选）
   DELETE /api/conversations/{id}  # 删除会话
   ```

2. **统计接口**
   ```python
   GET /api/stats/overview         # 概览统计（今日/本周/本月）
   GET /api/stats/trend            # 趋势数据（支持时间范围）
   GET /api/stats/models           # 模型使用分布
   ```

3. **统计服务**
   - 创建 `app/services/stats_service.py`
   - 支持按日/周/月聚合
   - 支持实时计算或预聚合

#### Step 4: Dashboard API 代理（server-go）

1. **新增 llm_proxy 接口**
   - 创建 `internal/application/usecase/llm_stats/`
   - 转发 llm-py 统计接口到 dashboard

2. **API 设计**
   ```go
   GET /api/v1/llm/stats/overview    # 转发 llm-py /api/stats/overview
   GET /api/v1/llm/stats/trend       # 转发 llm-py /api/stats/trend
   GET /api/v1/llm/conversations     # 转发 llm-py /api/conversations
   ```

3. **Handler 实现**
   - 创建 `internal/interfaces/http/handler/llm_stats_handler.go`
   - 统一响应格式封装

## 接口定义

### llm-py 接口

| 接口 | 方法 | 路径 | 请求参数 | 响应 |
|------|------|------|----------|------|
| 会话列表 | GET | /api/conversations | page, page_size, start_date, end_date, source | 分页会话列表 |
| 会话详情 | GET | /api/conversations/{id} | - | 会话详情+消息列表 |
| 消息列表 | GET | /api/messages | conversation_id, page, page_size | 分页消息列表 |
| 删除会话 | DELETE | /api/conversations/{id} | - | 删除结果 |
| 统计概览 | GET | /api/stats/overview | - | 今日/本周/本月统计数据 |
| 趋势数据 | GET | /api/stats/trend | days, group_by(day/week/month) | 趋势时间序列 |
| 模型分布 | GET | /api/stats/models | start_date, end_date | 模型使用分布 |

### server-go 接口（Dashboard 使用）

| 接口 | 方法 | 路径 | 请求参数 | 响应 |
|------|------|------|----------|------|
| LLM 概览 | GET | /api/v1/llm/stats/overview | - | 调用次数、token 消耗、平均延迟 |
| LLM 趋势 | GET | /api/v1/llm/stats/trend | days=7, group_by=day | 日期数组+调用次数数组 |
| 对话列表 | GET | /api/v1/llm/conversations | page, page_size | 会话列表 |
