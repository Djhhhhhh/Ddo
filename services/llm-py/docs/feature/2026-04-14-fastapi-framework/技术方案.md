# llm-py: FastAPI 框架 - 技术方案

## 背景与目标

llm-py 是 Ddo 项目的 LLM 代理服务，负责：
1. 代理 OpenRouter API 调用
2. 提供 RAG（检索增强生成）知识库服务
3. 为 server-go 提供 NLP 意图识别能力

**目标**: 搭建 FastAPI 基础框架，包含项目结构、依赖配置、核心模块初始化和基础健康检查接口。

## 需求概述

需要完成以下功能：
1. 创建标准 Python 项目结构
2. 配置 FastAPI + Uvicorn 运行环境
3. 实现模块化的 API 路由组织
4. 添加统一配置管理和日志系统
5. 提供 `/health` 健康检查端点

## 技术方案

### 方案选型

- **框架**: FastAPI (异步、自动文档、类型安全)
- **服务器**: Uvicorn (ASGI，支持热重载)
- **配置管理**: Pydantic Settings (环境变量 + 配置文件)
- **日志**: Python standard logging + structlog (可选)
- **Python版本**: 3.11+

### 核心实现步骤

1. **项目结构初始化**
   - 创建 `app/` 主包目录
   - 创建 `app/api/` API路由目录
   - 创建 `app/core/` 核心业务逻辑目录
   - 创建 `app/utils/` 工具函数目录

2. **依赖配置**
   - 创建 `requirements.txt` 列出核心依赖
   - 包含: fastapi, uvicorn[standard], pydantic-settings, python-dotenv

3. **配置系统**
   - 在 `app/core/config.py` 实现 Pydantic Settings
   - 支持从环境变量和 `.env` 文件读取
   - 配置项: 服务端口、主机、OpenRouter API Key、日志级别

4. **FastAPI 主应用**
   - 在 `app/main.py` 创建 FastAPI 实例
   - 配置 CORS 中间件
   - 挂载各模块路由 (chat, models, health, nlp, rag)
   - 实现 lifespan 事件 (启动/关闭)

5. **健康检查 API**
   - 在 `app/api/health.py` 实现 `/health` 端点
   - 返回 status 和 version
   - 后续扩展: 检测 OpenRouter 连接状态

6. **API 模块结构**
   - `app/api/chat.py`: Chat Completion 路由 (占位)
   - `app/api/models.py`: 模型管理路由 (占位)
   - `app/api/health.py`: 健康检查路由
   - `app/api/nlp.py`: NLP 路由 (占位)
   - `app/api/rag.py`: RAG 路由 (占位)
   - `app/api/__init__.py`: 路由聚合

7. **启动脚本**
   - 创建 `main.py` 项目根目录入口
   - 使用 Uvicorn 启动服务
   - 默认绑定 `127.0.0.1:8000`

## 接口定义

| 接口 | 方法 | 路径 | 请求参数 | 响应 |
|------|------|------|----------|------|
| 健康检查 | GET | /health | 无 | `{ "status": "ok", "version": "0.1.0" }` |
