# Web: Knowedge 信息页面 - 技术方案

## 背景与目标

用户目前在 CLI 中通过 `ddo kb` 命令管理知识库，但无法直观地查看已录入的知识库内容概览。本任务目标是实现一个知识库信息展示页面，让用户可以直观地了解当前知识库的状态、内容分类、条目数量等信息，作为 Dashboard 的补充视图。

同时，右侧提供一个**知识库测试对话界面**，让用户可以直接在页面上测试 RAG 问答能力，快速验证知识库内容的检索效果。

根据 mvp.md，Web 端定位是弥补 CLI 的可视化短板，知识库部分仅支持查看和搜索。

## 需求概述

### 左侧：知识库信息展示
1. **知识库概览卡片**：展示总条目数、本月新增、存储占用
2. **知识库列表**：分页展示知识库条目（标题、内容摘要、分类、标签、创建时间）
3. **搜索功能**：关键词搜索，支持标题和内容匹配
4. **分类筛选**：按分类筛选知识条目
5. **详情查看**：点击条目查看完整内容

### 右侧：知识库测试对话
1. **RAG 问答界面**：输入问题，调用 RAG 引擎获取答案
2. **会话历史**：显示当前会话的问题和回答
3. **引用来源**：显示回答引用的知识条目
4. **清空会话**：重置对话历史

## 技术方案

### 方案选型
- **前端框架**：Vue 3 + TypeScript（已有）
- **HTTP 客户端**：已有 Axios 封装（`src/api/knowledge.ts`）
- **路由**：新增 `/knowledge` 路由
- **UI 组件**：基于 Ollama 设计系统（DESIGN.md）- 纯灰度、Pill 按钮、零阴影、12px/9999px 二元 border-radius
- **布局**：左右分割布局（CSS Grid 或 Flexbox）

### DESIGN.md 样式规范要点
| 元素 | 规范 |
|------|------|
| 页面背景 | Pure White (`#ffffff`) |
| 主文字 | Pure Black (`#000000`) |
| 次要文字 | Stone (`#737373`) |
| 按钮背景 | Light Gray (`#e5e5e5`) |
| 边框 | Light Gray (`#e5e5e5`) |
| 容器圆角 | 12px（非交互元素） |
| 交互元素圆角 | 9999px（Pill 形状） |
| 阴影 | 零阴影 |
| 字重 | 400（正文）或 500（标题） |

## 接口定义

### 知识库信息接口

| 接口 | 方法 | 路径 | 请求参数 | 响应 |
|------|------|------|----------|------|
| listKnowledge | GET | /api/v1/knowledge | page, page_size | { code, message, data: { total, items: [...] } } |
| searchKnowledge | GET | /api/v1/knowledge/search | q (关键词) | { code, message, data: [...] } |
| getKnowledge | GET | /api/v1/knowledge/:uuid | - | { code, message, data: { knowledge: {...} } } |
| deleteKnowledge | POST | /api/v1/knowledge/:uuid/delete | - | { code, message, data: { success } } |

### 知识库测试接口

| 接口 | 方法 | 路径 | 请求参数 | 响应 |
|------|------|------|----------|------|
| askKnowledge | POST | /api/v1/knowledge/ask | { question, top_k } | { code, message, data: { answer, sources: [...] } } |
