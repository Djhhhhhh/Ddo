# 项目图标统一 - 技术方案

## 背景与目标

当前 web 端的项目图标存在不一致问题，需要将所有的项目 icon 统一使用 `services/web-ui/public/icons/icon.svg`，包括任务栏托管图标。

## 需求概述

1. 统一所有项目 icon 引用路径
2. 确保任务栏托管图标也使用同一图标文件
3. 删除冗余的图标文件，简化资源管理

## 技术方案

### 方案选型

- **技术选型**: Vite + React 项目的静态资源管理
- **设计模式**: 单一资源源（Single Source of Truth）
- **工具**: 文件系统操作、配置修改

### 核心实现步骤

1. **查找所有图标引用**
   - 搜索项目中所有使用 `.ico`, `.png`, `.svg` 作为图标的地方
   - 识别 `favicon`, `apple-touch-icon`, `manifest.json` 等引用点

2. **更新 HTML 和配置**
   - 修改 `index.html` 中的 favicon 链接
   - 更新 `manifest.json` 中的图标配置
   - 修改 PWA 相关配置

3. **更新任务栏图标**
   - 查找任务栏托管相关的组件
   - 确保使用统一的 icon.svg

4. **清理冗余文件**
   - 删除不再使用的图标文件
   - 确保 `public/icons/icon.svg` 存在且可用

## 目录结构

```
services/web-ui/
├── public/
│   └── icons/
│       └── icon.svg          # 统一使用的图标
├── index.html                # 更新 favicon 引用
└── manifest.json             # 更新图标配置
```
