# Electron 通知客户端 - 技术方案

## 背景与目标

CLI 作为核心主体，缺少弹出系统通知的能力。通过 Electron 实现通知客户端（`ddo ding` 命令启动），作为 CLI 的扩展渲染层，弥补无法弹窗通知的短板。

目标用户：使用 CLI 的开发者，需要系统通知或灵动岛提示作为操作入口的补充。

## 需求概述

实现一个 Electron 桌面客户端，通过 `ddo ding` 命令启动，连接后端服务，接收定时任务触发事件，以灵动岛悬浮窗口和系统通知中心两种形式展示通知。

## 技术方案

### 方案选型

- **技术选型**：Electron 28+（支持 ESM）+ TypeScript
- **设计模式**：主进程 + 预加载脚本 + 渲染进程（Vue）三层架构
- **依赖库/工具**：
  - `electron` 28+
  - `electron-builder`（打包）
  - `electron-store`（本地配置存储）
  - `electron-log`（日志记录）

### 核心实现步骤

#### 1. 初始化 Electron 项目结构

在 `web-ui/electron/` 目录下创建：

```
electron/
├── main.ts                  # 主进程入口
├── preload.ts               # 预加载脚本（桥接主进程和渲染进程）
├── windows/
│   ├── mainWindow.ts       # 主窗口（可选配置）
│   └── islandWindow.ts     # 灵动岛悬浮窗口
├── tray.ts                  # 系统托盘管理
├── notification.ts          # 通知管理（灵动岛 + 系统通知）
├── ipc.ts                   # IPC 通信定义
└── store.ts                 # 本地配置存储
```

#### 2. 实现主进程（main.ts）

- 创建 BrowserWindow 实例
- 加载 Web UI（http://localhost:3000）
- 管理窗口生命周期
- 注册系统托盘
- 启用全局快捷键（显示/隐藏窗口）
- 连接后端 WebSocket 或轮询通知接口

#### 3. 实现系统托盘（tray.ts）

- 创建 Tray 图标
- 右键菜单：显示窗口、设置、退出
- 单击图标显示/隐藏主窗口
- 托盘图标根据通知状态变化（如有新通知时变化）

#### 4. 实现灵动岛悬浮窗口（islandWindow.ts）

- 创建小型 BrowserWindow（无标题栏、置顶、透明度）
- 位置：屏幕右上角
- 样式：圆角卡片，阴影，自动消失（5秒）
- 支持快捷操作按钮："完成"、"5分钟后提醒"、"查看详情"
- 点击窗口可打开详情或执行操作

#### 5. 实现通知管理（notification.ts）

- **灵动岛通知**：调用 islandWindow 显示悬浮卡片
- **系统通知**：调用 Electron Notification API
- 通知级别配置（普通、重要、紧急）
- 通知历史记录存储

#### 6. 实现预加载脚本（preload.ts）

- 暴露安全的 IPC 通信接口
- 通知事件转发到渲染进程
- 安全上下文隔离

#### 7. 实现 Vue 端通知组件

在 `src/components/DingIsland/` 目录下：

```
components/DingIsland/
├── DingIslandCard.vue      # 灵动岛卡片组件
└── DingIslandStore.ts      # Pinia 通知状态管理
```

#### 8. CLI 集成

- 在 CLI 添加 `ddo ding` 命令
- 命令启动 Electron 进程
- 通过 `ddo ding` 命令行参数传递配置（如端口）

#### 9. 通知事件接收

- Electron 启动后连接 `ws://localhost:8080/ws/notify`
- 或轮询 `GET /api/notifications/subscribe`
- 接收通知后根据类型显示灵动岛或系统通知

## 接口定义

### Electron 与后端通信

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 通知订阅 | WebSocket | /ws/notify | 实时接收通知事件 |
| 通知轮询 | GET | /api/notifications/subscribe | 备用轮询方案 |

### IPC 通信（主进程 ↔ 渲染进程）

| 通道 | 方向 | 说明 |
|------|------|------|
| notification:show | main → renderer | 显示通知到 Vue 组件 |
| notification:action | renderer → main | 用户点击通知操作 |
| window:toggle | main | 显示/隐藏主窗口 |
