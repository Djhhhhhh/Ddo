# Web: API 对接 - 技术方案

## 背景与目标

根据 MVP 需求文档，Web 端需要与 server-go 后端进行 API 通信，实现 Dashboard 数据展示和表单配置功能。Vue 端已有 Axios 封装基础（`src/api/client.ts`），但具体 API 函数尚未创建。

目标：创建完整的 API 模块，对接 server-go 的 REST API，为 Dashboard 和配置页面提供数据支持。

## 需求概述

根据 server-go 路由定义和 MVP 功能列表，需要实现以下 API 模块：

| API 模块 | 用途 | 对应页面 |
|----------|------|----------|
| health.ts | 服务健康状态 | Dashboard |
| metrics.ts | 综合指标统计 | Dashboard |
| knowledge.ts | 知识库 CRUD | 配置页面 |
| timer.ts | 定时任务 CRUD | 配置页面 |
| mcp.ts | MCP 配置 CRUD | MCP 页面 |
| settings.ts | 系统设置 | 配置页面 |

## 接口定义

### server-go API 路由（已实现）

| 接口 | 方法 | 路径 | 请求参数 | 响应 |
|------|------|------|----------|------|
| 健康检查 | GET | /health | - | `{code, message, data: {status, version, mysql, badgerdb}}` |
| 综合指标 | GET | /api/v1/metrics | - | `{code, message, data: {status, version, services, timers, knowledge, mcps}}` |
| 知识库列表 | GET | /api/v1/knowledge | - | `{code, message, data: [{uuid, title, content, category, tags}]}` |
| 知识库搜索 | GET | /api/v1/knowledge/search | ?q=keyword | `{code, message, data: [{uuid, title, snippet}]}` |
| 定时任务列表 | GET | /api/v1/timers | - | `{code, message, data: [{uuid, name, cron, status, next_run}]}` |
| MCP 列表 | GET | /api/v1/mcps | - | `{code, message, data: [{uuid, name, type, status, config}]}` |
| MCP 测试 | POST | /api/v1/mcps/:uuid/test | - | `{code, message, data: {success, tools, error}}` |

## 技术方案

### 方案选型

- **HTTP 客户端**：Axios 1.0+（已封装在 client.ts）
- **API 设计**：每个领域一个独立文件，统一导出
- **类型定义**：使用 TypeScript interface 定义请求/响应类型
- **错误处理**：统一错误处理，在 client.ts 拦截器中处理

### 核心实现步骤

1. **创建 TypeScript 类型定义**
   - 创建 `src/api/types.ts` - 定义通用响应类型和领域类型

2. **创建领域 API 模块**
   - 创建 `src/api/metrics.ts` - 综合指标（Dashboard）
   - 创建 `src/api/knowledge.ts` - 知识库
   - 创建 `src/api/timer.ts` - 定时任务
   - 创建 `src/api/mcp.ts` - MCP 配置
   - 创建 `src/api/settings.ts` - 系统设置

3. **统一导出**
   - 在 `src/api/index.ts` 导出所有 API 函数

## API 函数签名设计

### metrics.ts
```typescript
export interface ServicesMetrics { server_go: string; llm_py: string; mysql: string }
export interface TimerMetrics { total: number; active: number }
export interface KnowledgeMetrics { total: number }
export interface McpMetrics { total: number }
export interface MetricsData {
  status: string; version: string
  services: ServicesMetrics; timers: TimerMetrics
  knowledge: KnowledgeMetrics; mcps: McpMetrics
}
export interface MetricsResponse { code: number; message: string; data: MetricsData }

export function getMetrics(): Promise<MetricsResponse>
```

### timer.ts
```typescript
export interface Timer { uuid: string; name: string; cron: string; status: string; next_run?: string }
export interface TimerListResponse { code: number; message: string; data: Timer[] }

export function listTimers(): Promise<TimerListResponse>
export function pauseTimer(uuid: string): Promise<void>
export function resumeTimer(uuid: string): Promise<void>
```

### mcp.ts
```typescript
export interface MCP { uuid: string; name: string; type: string; status: string; config: object }
export interface MCPListResponse { code: number; message: string; data: MCP[] }
export interface MCPTestResponse { code: number; message: string; data: { success: boolean; tools: string[]; error?: string } }

export function listMCPs(): Promise<MCPListResponse>
export function testMCP(uuid: string): Promise<MCPTestResponse>
```
