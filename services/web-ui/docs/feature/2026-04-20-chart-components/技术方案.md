# Web: 图表组件 - 技术方案

## 背景与目标

Dashboard 目前只能展示静态的服务状态卡片和统计数据，缺少数据可视化能力。本任务目标是实现图表组件，用于展示 LLM 调用趋势、定时任务统计等数据的可视化呈现，提升 Dashboard 的可读性和用户体验。

根据 mvp.md，Dashboard 需要支持 LLM 调用趋势图表（近7天调用量柱状图/折线图）以及定时任务统计图表。

## 需求概述

### 图表类型
1. **LLM 调用趋势图**：近7天调用量柱状图或折线图，显示总调用数、成功率、平均延迟
2. **定时任务统计**：总任务数、运行中/暂停数、今日执行次数、成功/失败数量（饼图或柱状图）
3. **知识库统计**：总条目数、本月新增、存储占用（饼图）
4. **Dashboard 自动刷新**：每 30 秒自动刷新，显示最后更新时间

## 技术方案

### 方案选型
- **图表库**：ECharts 5.x（轻量、功能丰富、支持 Vue 封装）
- **Vue 封装**：自定义 `Chart.vue` 组件，接收配置和数据进行渲染
- **响应式**：窗口 resize 时自动调整图表大小
- **主题支持**：跟随 Web UI 主题（dark/light）切换图表配色

### 核心实现步骤

#### 1. 安装 ECharts

```bash
cd services/web-ui
npm install echarts vue-echarts
```

#### 2. 创建 Chart 组件（`src/components/Chart/BaseChart.vue`）

**功能点**：
- 基于 ECharts 封装
- 接收 `option`（ECharts 配置）和 `data`（图表数据）props
- 响应式容器，自动适应父容器宽度
- 主题跟随系统（dark/light）
- 加载状态和空数据状态处理

**组件接口**：
```typescript
interface Props {
  option: EChartsOption  // ECharts 配置对象
  loading?: boolean        // 加载状态
  height?: string          // 图表高度，默认 '300px'
}
```

#### 3. 创建预设图表组件

**LineChart.vue**（LLM 调用趋势）：
- 接收 `xAxisData`（日期数组）和 `seriesData`（调用量数组）
- 支持多条折线（调用量、成功数、失败数）
- 鼠标悬停显示数值 tooltip

**BarChart.vue**（定时任务统计）：
- 接收分类数据和数值
- 支持水平/垂直柱状图模式

**PieChart.vue**（知识库占比）：
- 接收 `{ name, value }` 格式的数据数组
- 支持标签显示百分比

#### 4. 集成到 Dashboard（`src/views/Dashboard/Dashboard.vue`）

**修改点**：
- 导入图表组件
- 调用 Metrics API 获取数据
- 将数据转换为图表配置格式
- 添加自动刷新逻辑（30秒轮询）

#### 5. 扩展 Metrics API（如需要）

检查 `services/server-go/internal/interfaces/http/handler/metrics_handler.go` 确认是否需要新增接口。

|------|------|------|------|
| metrics | GET | /api/v1/metrics | { code, message, data: { status, version, services, timers, knowledge, mcps } } |

注：现有 `/api/v1/metrics` 接口已可获取基础统计数据，图表组件直接复用该接口数据。
