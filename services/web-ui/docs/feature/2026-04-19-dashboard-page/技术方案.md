# Web: Dashboard 页面 - 技术方案

## 背景与目标

根据 MVP 需求文档，Dashboard 是 Web 端的核心页面，用于将 CLI `ddo status` 的文本信息可视化展示。目标是弥补 CLI 的可视化短板，提供服务健康状态卡片、定时任务统计、知识库统计等监控能力。

当前状态：已有基础的 Card 组件和 DashboardView.vue 框架，但数据为 placeholder，需对接真实 API 并完善图表组件。

## 需求概述

### Dashboard 页面功能点

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 服务状态卡片 | P0 | CLI、server-go、llm-py、MySQL 四个状态卡片，显示状态指示灯、版本号、uptime |
| 定时任务统计 | P0 | 总任务数、运行中/暂停数、今日执行次数、成功/失败数量 |
| 知识库统计 | P0 | 总条目数、本月新增、存储占用（MB） |
| MCP 状态列表 | P0 | MCP 名称、类型、状态、最后检查时间、错误状态标记 |
| LLM 调用趋势 | P1 | 近7天调用量柱状图/折线图，显示总调用数、成功率、平均延迟 |
| 自动刷新 | P1 | 每 30 秒自动刷新，显示最后更新时间，支持手动刷新按钮 |

## 技术方案

### 方案选型

- **技术选型**：Vue 3 + TypeScript + Vite + TailwindCSS + Pinia
- **图表库**：Chart.js（轻量级，支持 Vue 3）
- **状态管理**：Pinia 管理 Dashboard 数据和自动刷新逻辑
- **设计模式**：遵循 Ollama 设计系统（纯灰度、Pill 按钮、12px border-radius）

### 核心实现步骤

1. **创建 API 模块**
   - 创建 `src/api/health.ts` - 服务健康状态 API
   - 创建 `src/api/knowledge.ts` - 知识库统计 API
   - 创建 `src/api/timer.ts` - 定时任务统计 API
   - 创建 `src/api/mcp.ts` - MCP 状态 API

2. **创建 Dashboard Store**
   - 创建 `src/stores/dashboard.ts` - 管理 Dashboard 数据和刷新逻辑

3. **创建 UI 组件**
   - 创建 `src/components/ServiceCard.vue` - 服务状态卡片组件
   - 创建 `src/components/StatCard.vue` - 统计卡片组件
   - 创建 `src/components/Charts/LineChart.vue` - 折线图组件
   - 创建 `src/components/Charts/BarChart.vue` - 柱状图组件

4. **完善 DashboardView.vue**
   - 对接真实 API 数据
   - 实现服务状态卡片网格
   - 实现统计卡片
   - 实现 MCP 状态列表
   - 实现 LLM 调用趋势图表
   - 实现自动刷新逻辑（30秒间隔）

## 接口定义

根据 server-go 现有 API 路由定义：

| 接口 | 方法 | 路径 | 响应数据结构 |
|------|------|------|-------------|
| 健康检查 | GET | /health | `{status: "ok", services: {...}}` |
| 知识库统计 | GET | /api/kb/stats | `{total: number, monthly_new: number, storage_mb: number}` |
| 定时任务列表 | GET | /api/timer/list | `{tasks: [{id, name, cron, status, next_run}]}` |
| MCP 列表 | GET | /api/mcp/list | `{mcps: [{id, name, type, status, last_check}]}` |
