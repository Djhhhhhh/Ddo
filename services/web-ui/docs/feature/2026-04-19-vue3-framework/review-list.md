## 功能验证清单

### Step 1: 项目初始化
- [ ] `npm install` 依赖安装成功 - 预期：所有依赖安装完成，无错误
- [ ] `npm run dev` 启动开发服务器 - 预期：控制台显示 "Vite dev server running at localhost:3000"

### Step 2: 路由跳转
- [ ] 访问 `http://localhost:3000/#/` - 预期：显示 Help 首页
- [ ] 访问 `http://localhost:3000/#/dashboard` - 预期：显示 Dashboard 页面
- [ ] 访问 `http://localhost:3000/#/mcp` - 预期：显示 MCP 页面
- [ ] 访问 `http://localhost:3000/#/timer` - 预期：显示 Timer 页面
- [ ] 访问 `http://localhost:3000/#/config` - 预期：显示 Config 页面
- [ ] 侧边栏菜单点击跳转 - 预期：页面正常切换

### Step 3: 主题切换
- [ ] 点击主题切换按钮（如果有）- 预期：页面在 light/dark/system 间切换
- [ ] 系统主题自动适配 - 预期：根据 OS 偏好自动切换

### Step 4: 顶部导航
- [ ] 导航链接显示 - 预期：Help、Dashboard、MCP、Timer、Config 五个链接
- [ ] 点击导航跳转 - 预期：页面跳转到对应路由
- [ ] 主题切换按钮 - 预期：点击在 light/dark/system 间切换

### Step 5: UI 组件样式
- [ ] 按钮组件 - 预期：Gray/White/Black 三种样式，Pill 形状（9999px border-radius）
- [ ] 卡片组件 - 预期：12px border-radius，无阴影，1px border
- [ ] 输入框 - 预期：Pill 形状，focus 有 ring 效果
- [ ] 整体灰度色系 - 预期：整个页面无彩色，只有灰色

### Step 6: API 代理
- [ ] 访问 `/health` 代理 - 预期：请求转发到 `127.0.0.1:8080/health`
- [ ] 访问 `/api/*` 代理 - 预期：请求转发到 `127.0.0.1:8080/api/*`

### Step 7: Help 首页内容
- [ ] 显示项目 Logo 和标题 - 预期：显示 "Ddo" 和副标题
- [ ] 显示核心功能介绍 - 预期：展示服务管理、知识库、定时任务等
- [ ] 显示 CLI 命令列表 - 预期：展示 `/help-web`、`/dashboard-web` 等命令

## 边界情况测试

- [ ] 刷新页面后路由保持 - 预期：Hash 路由保持不变
- [ ] 访问不存在的路由 - 预期：显示 404 或回退到首页
- [ ] server-go 未启动时访问 API - 预期：显示网络错误，不崩溃
