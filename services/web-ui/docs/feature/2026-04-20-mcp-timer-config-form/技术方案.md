# Web: MCP/Timer 配置表单 - 技术方案

## 背景与目标

CLI 的 MCP 和定时任务配置目前只能通过命令式操作完成（`ddo mcp create`、`ddo timer create`），用户体验较差。本任务目标是在 Web UI 中实现可视化的 MCP 和定时任务配置管理界面，让用户可以通过表单操作完成配置的创建、编辑、测试和删除。

根据 mvp.md，Web 端是 CLI 的扩展渲染层，定位是弥补 CLI 的可视化短板。

## 需求概述

### MCP 配置管理
- MCP 列表页：表格展示（名称、类型、状态）
- 新增 MCP 表单：支持名称、类型（stdio/http/sse）、动态配置
- MCP 测试功能：测试连接成功/失败，显示可用工具列表
- 删除 MCP：删除确认对话框

### 定时任务管理
- 定时任务列表页：表格展示（名称、cron、状态）
- 暂停/恢复功能：状态控制按钮
- 执行日志查看：点击任务查看执行历史

## 技术方案

### 方案选型
- **前端框架**：Vue 3 + TypeScript + Vite（已有）
- **UI 组件**：基于 Ollama 设计系统的自定义组件（Gray/White/Black Pill、12px border-radius）
- **HTTP 客户端**：已有 Axios 封装（`src/api/client.ts`）
- **路由管理**：已有 Vue Router 配置
- **图表库**：轻量级方案 ECharts 或 Chart.js（本任务不涉及图表渲染）

### 核心实现步骤

#### 1. 扩展 API 层（`src/api/mcp.ts`、`src/api/timer.ts`）

**MCP API 函数**：
```typescript
// 新增
export function createMCP(data: CreateMCPRequest): Promise<CreateMCPResponse>
export function getMCP(uuid: string): Promise<GetMCPResponse>
export function deleteMCP(uuid: string): Promise<DeleteMCPResponse>
```

**Timer API 函数**：
```typescript
// 新增
export function getTimer(uuid: string): Promise<GetTimerResponse>
export function createTimer(data: CreateTimerRequest): Promise<CreateTimerResponse>
export function updateTimer(uuid: string, data: UpdateTimerRequest): Promise<UpdateTimerResponse>
export function deleteTimer(uuid: string): Promise<DeleteTimerResponse>
export function triggerTimer(uuid: string): Promise<TriggerTimerResponse>
export function listTimerLogs(uuid: string): Promise<ListTimerLogsResponse>
```

#### 2. 创建 MCP 页面（`src/views/MCP/McpList.vue`）

**功能点**：
- 表格展示所有 MCP 配置（名称、类型、状态、上次测试时间）
- 新增 MCP 按钮 → 打开表单弹窗
- 测试按钮 → 调用 `testMCP` API，显示结果
- 删除按钮 → 确认对话框 → 调用 `deleteMCP` API

**表单字段**（根据 `CreateMCPRequest`）：
| 字段 | 类型 | 验证 |
|------|------|------|
| name | string | required |
| description | string | optional |
| type | select | required, oneOf: stdio/http/sse |
| command | string | type=stdio 时 required |
| args | string[] | optional |
| env | string[] | optional |
| url | string | type=http/sse 时 required |
| headers | object | optional |

#### 3. 创建定时任务页面（`src/views/Timer/TimerList.vue`）

**功能点**：
- 表格展示所有定时任务（名称、cron、状态、下次执行时间）
- 暂停/恢复按钮 → 调用 `pauseTimer`/`resumeTimer` API
- 查看日志按钮 → 弹出日志面板
- 触发按钮 → 手动触发一次

#### 4. 创建表单组件（`src/components/Form/McpForm.vue`、`src/components/Form/TimerForm.vue`）

**McpForm**：
- 根据 type 动态显示不同字段（stdio 显示 command/args/env，http/sse 显示 url/headers）
- 表单验证规则
- 提交时调用 `createMCP`

**TimerForm**：
- Cron 表达式输入（支持可视化 cron 选择器）
- 回调配置（URL、Method、Headers、Body）
- 时区选择

## 接口定义

### MCP 接口

| 接口 | 方法 | 路径 | 请求参数 | 响应 |
|------|------|------|----------|------|
| listMCP | GET | /api/v1/mcps | type, status, page, page_size | { code, message, data: { total, items: [...] } } |
| createMCP | POST | /api/v1/mcps | { name, description, type, command, args, env, url, headers } | { code, message, data: { uuid, name, type, status } } |
| getMCP | GET | /api/v1/mcps/:uuid | - | { code, message, data: { mcp: {...} } } |
| deleteMCP | POST | /api/v1/mcps/:uuid/delete | - | { code, message, data: { success } } |
| testMCP | POST | /api/v1/mcps/:uuid/test | timeout | { code, message, data: { status, tools, elapsed_ms, error } } |

### Timer 接口

| 接口 | 方法 | 路径 | 请求参数 | 响应 |
|------|------|------|----------|------|
| listTimers | GET | /api/v1/timers | status, page, page_size | { code, message, data: { total, items: [...] } } |
| createTimer | POST | /api/v1/timers | { name, cron_expr, timezone, callback_url, callback_method, callback_headers, callback_body } | { code, message, data: { uuid, name, status, next_run_at } } |
| getTimer | GET | /api/v1/timers/:uuid | - | { code, message, data: { timer: {...} } } |
| updateTimer | POST | /api/v1/timers/:uuid/update | { name, cron_expr, ... } | { code, message, data: { uuid, name, status } } |
| pauseTimer | POST | /api/v1/timers/:uuid/pause | - | { code, message, data: { uuid, status } } |
| resumeTimer | POST | /api/v1/timers/:uuid/resume | - | { code, message, data: { uuid, status } } |
| deleteTimer | POST | /api/v1/timers/:uuid/delete | - | { code, message, data: { success } } |
| triggerTimer | POST | /api/v1/timers/:uuid/trigger | - | { code, message, data: { uuid, status } } |
| listTimerLogs | GET | /api/v1/timers/:uuid/logs | status, page, page_size | { code, message, data: { total, items: [...] } } |
